# Ceph - 统一分布式存储
> 与NFS + nfs-provisioner / MinIO 的对比参考> 当前使用：NFS + MinIO

---

## 是什么
Ceph 是统一的分布式存储系统，同时提供 **块存储（RBD）**、**文件存储（CephFS）**、**对象存储（RGW）**三种接口。一个 Ceph 集群可以替代 NFS + MinIO + 本地盘，是生产环境中最流行的开源 SDS 方案。
---

## 与NFS / MinIO 的核心区别
| 维度 | NFS + MinIO | Ceph |
|------|-------------|------|
| **接口** | 文件（NFS） 对象（MinIO）| 块 + 文件 + 对象（统一）|
| **数据一致性 * | 最终一致性 | 强一致性 |
| **高可用 * | NFS 单点 / MinIO 分布 | 是全分布式，自我修复 |
| **性能** | NFS 低，MinIO 高 | 高（SSD 优化，万兆网络）|
| **运维** | 无 | 极高 |
| **硬件需求 * | 一般机器 | 推荐 SSD + 万兆网络 |

---

## 引入 Ceph 你能得到什么
| 收益 | 说明 |
|------|------|
| ✅ 统一存储 | 一套集群提供块/文件/对象，不必维护多套系统 |
| ✅ 强一致性| 数据写入多副本后返回，不用担心不一致性 |
| ✅ 自我修复 | 检测到磁盘/节点故障后自动恢复数据|
| ✅ 弹性扩缩| 新增 OSD 节点自动扩存储和性能 |

## 引入 Ceph 的代价
| 代价 | 说明 |
|------|------|
| ⛔ 运维极复杂| Ceph 是业界公认运维最困难的存储之一，需要专职人员 |
| ⛔ 硬件要求高| 没有 SSD + 万兆网，Ceph 性能会很差 |
| ⛔ 资源消耗大 | Ceph 进程自身消耗 CPU/内存，小规模不划算 |
| ⛔ 部署周期长| 一套生产级 Ceph 从规划到稳定运行可能需要数月|
