## 替代方案一览

> 现有中间件的替代工具对比调研，放在各分类 `未采用/` 目录中。每个工具说明与当前方案的核心区别、引入收益和代价，供后续选型参考。

| 当前中间件 | 替代方案 | 核心差异 | 引入能获得什么 |
|-----------|---------|---------|----------------|
| **Nginx** | [Kong](./03-网关/未采用/Kong.md) | 插件生态 + Dashboard + 热生效 | API 管理界面、开箱即用的限流/鉴权、灰度发布 |
| **Nginx** | [APISIX](./03-网关/未采用/APISIX.md) | 国产高性能，路由算法优 | 热加载不中断、多集群统一管理 |
| **Nginx** | [Envoy](./03-网关/未采用/Envoy.md) | 服务网格数据面 | 原生 gRPC、内置 tracing、动态 xDS 配置 |
| **Nginx** | [Traefik](./03-网关/未采用/Traefik.md) | K8s 自动服务发现 | 零配置部署、自动 HTTPS、内置中间件 |
| **Redis** | [Dragonfly](./01-数据库/未采用/Dragonfly.md) | 多线程架构，单实例百万 QPS | 5x+ 吞吐、内存省 30%、架构简化 |
| **Kafka** | [Pulsar](./02-消息队列/未采用/Pulsar.md) | 存算分离 + 原生多租户 | 弹性扩缩无感、命名空间隔离 |
| **RabbitMQ** | [NATS](./02-消息队列/未采用/NATS.md) | 微秒级延迟 + 百万 QPS | 极低延迟、极高吞吐、极简运维 |
| **Prometheus** | [VictoriaMetrics](./04-可观测性/未采用/VictoriaMetrics.md) | 高性能 TSDB，存储压缩率高 | 省 2-3 倍存储空间、查询更快、长期存储内置 |
| **Loki** | [Elasticsearch](./04-可观测性/未采用/Elasticsearch.md) | 全文索引 + 聚合分析 | 全文检索、Kibana 可视化、Machine Learning |
| **HAMI** | [Volcano](./05-AI基础设施/未采用/Volcano.md) | 批调度 + Gang-Scheduling | 分布式训练必备、队列配额、DAG 任务编排 |
| **Sandbox** | [gVisor](./05-AI基础设施/未采用/gVisor.md) | 系统调用拦截沙箱 | 更强安全隔离，防容器逃逸 |
| **NFS** | [Longhorn](./06-部署与存储/未采用/Longhorn.md) | 分布式块存储 + 多副本 | 高可用卷、秒级快照、GUI 管理 |
| **NFS/MinIO** | [Ceph](./06-部署与存储/未采用/Ceph.md) | 统一块/文件/对象存储 | 一套集群替代 NFS + MinIO + 本地盘 |
| **GitLab** | [Gitea](./06-部署与存储/未采用/Gitea.md) | 极轻量 Git 服务 | 1C2G 能跑、升级替换二进制即可、速度极快 |
| **Harbor** | [Nexus](./06-部署与存储/未采用/Nexus.md) | 全品类制品仓库 | Maven/npm/PyPI/Docker 统一管理、代理中央仓库 |
