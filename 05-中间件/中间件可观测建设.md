# 中间件可观测建设

> 针对现有全部中间件，建立统一的指标采集、日志收集、告警响应体系。
> 基于 Prometheus + Loki + Grafana 栈，覆盖每个中间件的可观测性需求。

---

## 一、总体架构

```
┌─────────────────────────────────────────────────────────┐
│                      Grafana                            │
│            Dashboard（可视化）+ Alerting（告警）           │
└────────┬────────────────────────────────┬────────────────┘
         │  PromQL                         │  LogQL
    ┌────▼────┐                     ┌──────▼──────┐
    │Prometheus│                     │    Loki     │
    │ 指标存储 │                     │  日志存储   │
    └──┬───┬──┘                     └──────┬───────┘
       │   │                               │
  Pull │   │ Push                    Push   │ Push
       │   │                               │
┌──────▼───▼──┐                     ┌──────▼───────┐
│  Exporters  │                     │  Promtail    │
│  (指标暴露) │                     │  (日志采集)   │
└──────┬───┬──┘                     └──────┬───────┘
       │   │                               │
  ┌────▼───▼───────────────────────────────▼────┐
  │              中间件实例                       │
  │  PG  Milvus  Redis  Kafka  RMQ  Nginx  ...  │
  └──────────────────────────────────────────────┘
```

### 2.3 告警规则

| 中间件 | 告警项 | 条件 | 严重度 |
|--------|-------|------|--------|
| **PostgreSQL** | 实例宕机 | `pg_up == 0` | critical |
| | 连接数过高 | `pg_stat_activity_count > max_connections * 0.8` | warning |
| | 复制延迟 | `pg_replication_lag > 10s` | warning |
| | 长事务 | `max pg_stat_activity_duration > 30min` | warning |
| **Milvus** | 搜索延迟 P99 > 1s | histogram_quantile(0.99, ...) > 1 | warning |
| | 数据节点宕机 | `up{component="datanode"} == 0` | critical |
| **Redis** | 实例宕机 | `redis_up == 0` | critical |
| | 内存 > 80% | `redis_memory_used / redis_memory_max * 100 > 80` | warning |
| | 命中率 < 90% | 缓存命中率 < 90 | warning |
| | 复制延迟 > 1s | `redis_replication_lag > 1` | warning |
| **Kafka** | Broker 宕机 | `kafka_broker_info == 0` | critical |
| | 消费者积压 | `kafka_consumer_group_lag > 10000` | warning |
| | 分区未同步 | `kafka_topic_partition_under_replicated > 0` | critical |
| | 磁盘 > 80% | `kafka_log_log_size / limit * 100 > 80` | warning |
| **RabbitMQ** | 节点宕机 | `rabbitmq_node_up == 0` | critical |
| | 队列积压 | `rabbitmq_queue_messages_ready > 10000` | warning |
| | 无消费者 | `rabbitmq_queue_consumers == 0` | warning |
| | 内存告警 | `rabbitmq_alarms_running{type="memory"} == 1` | critical |
| **Nginx** | 5xx 率 > 5% | `rate(5xx) / rate(all) * 100 > 5` | warning |
| | 请求延迟 P99 > 3s | histogram_quantile(0.99, ...) > 3 | warning |
| **MinIO** | 节点离线 | `minio_cluster_nodes_offline_total > 0` | critical |
| | 磁盘 > 85% | `minio_disk_used / total * 100 > 85` | warning |
| **GPU** | GPU 异常温度 | `DCGM_FI_DEV_GPU_TEMP > 85` | warning |
| | GPU 显存 OOM | `DCGM_FI_DEV_FB_USED / FB_TOTAL * 100 > 95` | critical |
| **Docker** | 容器频繁重启 | `rate(container_restarts[15m]) > 3` | warning |
| **K8s** | Pod  CrashLoop | `kube_pod_container_status_restarts > 3` | warning |
| | 节点 NotReady | `kube_node_status_condition{condition="Ready",status="true"} == 0` | critical |

## 三、一站式部署

### 3.1 Prometheus Stack（含 kube-state-metrics + node-exporter）

```bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm upgrade --install kube-prometheus-stack prometheus-community/kube-prometheus-stack \
  --namespace observability --create-namespace \
  --set prometheus.prometheusSpec.retention=30d \
  --set prometheus.prometheusSpec.storageSpec.volumeClaimTemplate.spec.storage=100Gi \
  --set prometheus.prometheusSpec.resources.requests.memory=4Gi \
  --set alertmanager.config.global.resolve_timeout=5m \
  --set grafana.adminPassword=<your-password>
```

### 3.2 批量部署各中间件 Exporter

将以下所有 ServiceMonitor 和应用部署通过一个 values 文件统一管理：

```yaml
# middleware-observability-values.yaml
postgres-exporter:
  enabled: true
  config:
    datasource: "postgresql://monitor:password@postgres:5432/postgres?sslmode=disable"

redis-exporter:
  enabled: true
  redisAddress: "redis://redis:6379"

kafka-exporter:
  enabled: true
  kafkaServers:
    - kafka:9092

rabbitmq-exporter:
  enabled: true
  rabbitmq:
    url: "http://rabbitmq:15672"
    username: admin
    password: <password>
```

使用 Prometheus-community 的 Helm Chart 批量安装：

```bash
# PostgreSQL Exporter
helm install postgres-exporter prometheus-community/prometheus-postgres-exporter \
  --namespace observability \
  --set config.datasource="postgresql://monitor:pass@pg:5432/postgres?sslmode=disable"

# Redis Exporter
helm install redis-exporter prometheus-community/prometheus-redis-exporter \
  --namespace observability \
  --set redisAddress="redis://:password@redis:6379"

# Kafka Exporter
helm install kafka-exporter prometheus-community/prometheus-kafka-exporter \
  --namespace observability \
  --set kafkaServers[0]=kafka:9092

# RabbitMQ 自带 Prometheus 插件（只需开启）
rabbitmq-plugins enable rabbitmq_prometheus
# 或 Helm values: metrics.enabled=true

# Nginx Exporter（需编译 nginx 时加 stub_status 或 vts 模块）
helm install nginx-exporter prometheus-community/prometheus-nginx-exporter \
  --namespace observability \
  --set nginx.scrapeUri="http://nginx:80/nginx_status"
```

### 3.3 Loki + Promtail 统一日志采集

```bash
helm upgrade --install loki grafana/loki-stack \
  --namespace observability \
  --set grafana.enabled=false \
  --set prometheus.enabled=false \
  --set loki.persistence.enabled=true \
  --set loki.persistence.size=100Gi \
  --set promtail.enabled=true
```

## 五、告警统一管理

### 5.1 AlertManager 配置

```yaml
# alertmanager-config.yaml
global:
  resolve_timeout: 5m
  wechat_api_url: "https://qyapi.weixin.qq.com/cgi-bin/"
  wechat_api_secret: <secret>

route:
  receiver: feishu
  group_by: [alertname, severity]
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  routes:
    - match:
        severity: critical
      receiver: feishu
      repeat_interval: 1h
    - match:
        severity: warning
      receiver: feishu
      repeat_interval: 4h

receivers:
  - name: feishu
    webhook_configs:
      - url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
        send_resolved: true
```

### 5.2 告警分级

| 级别 | 响应要求 | 通知方式 | 示例 |
|------|---------|---------|------|
| **P0 (critical)** | 立即处理，15min 内响应 | 飞书 @所有人 + 电话 | 数据库宕机、K8s 节点 NotReady |
| **P1 (warning)** | 当日处理 | 飞书群通知 | 磁盘 > 80%、消费者积压 |
| **P2 (info)** | 观察或排期 | 飞书消息（不艾特）| 缓存命中率降低、慢查询增多 |

```yaml
事项:
  - Milvus: 自建 Dashboard + 搜索延迟告警

  - Sandbox/Firecrawl: 自定义 metrics 端点

  - 业务指标: 应用层暴露 /metrics（请求量、错误率、延迟）
```
