## 推荐引入

### Dashboard 清单

| Dashboard | 内容 | 数据源 | 来源 |
|-----------|------|--------|------|
| 全局概览 | 服务健康度、核心 RED 指标 | Prometheus | 自建 |
| K8s 集群 | 节点/Pod/资源 | Prometheus | 导入 315、6417 |
| AI 推理服务 | GPU 显存、TTFT、Token 用量 | Prometheus | 导入 21766 + 自建 |
| 中间件 | Redis/MQ/DB 状态 | Prometheus | 各 Exporter |
| 应用日志 | 日志搜索 | Loki | 自建 |
| 链路追踪 | 请求调用链 | Tempo | 自建 |

### 告警分层体系

| 级别 | 响应 | 通知方式 |
|------|------|---------|
| P0 | 立即（5 分钟无响应升级）| 飞书 + 短信 + 电话 |
| P1 | 30 分钟内 | 飞书 + 短信 |
| P2 | 工作时间处理 | 飞书通知 |
| P3 | 下次迭代 | 邮件/工单 |

### 迁移路径

```
当前: Zabbix 告警 → 飞书
      Prometheus → Alertmanager（未配置）

短期: Zabbix 告警 → 飞书（并行）
      Alertmanager → 飞书（新增，覆盖应用/AI 告警）

长期: Zabbix 退居纯硬件告警，业务告警全部迁移到 Alertmanager
```
