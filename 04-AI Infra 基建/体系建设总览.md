# AI Infra 基建 体系建设总览

> AI Infra 基建是 SmartVision 产品差异化的核心——GPU 资源是否被充分利用、推理服务是否稳定高效、模型资产是否有治理，直接决定 AI 功能的交付速度和服务质量。本文是 AI Infra 领域的现状分析与建设规划总纲。

---

## 一、现状与痛点

### 1.1 四个核心子域评估

| 子域 | 当前状态 | 主要痛点 | 影响 |
|------|---------|---------|------|
| **GPU 可观测** | ?? DCGM 未部署 | 无 SM 利用率/显存/温度时序指标，GPU 空跑无法主动发现 | 每月浪费大量 GPU 算力 |
| **GPU 调度** | ?? HAMI 已部署 | 宿主机裸进程绕过 HAMI 管控，Quota 形同虚设 | 资源争抢时无依据，故障难定位 |
| **推理服务** | ?? vLLM 多实例运行 | 无统一部署规范，版本分散，参数靠经验试调 | 性能不稳定，运维成本高 |
| **AI 模型治理** | ?? 无 | 模型版本无记录，无 A/B 测试框架，模型来源不可追溯 | 生产模型出问题无法快速回滚 |

### 1.2 典型问题场景

**场景一：GPU 空跑无人知晓**
某 vLLM 实例长期处于等待请求状态，GPU SM 利用率 < 5%，因无监控从未主动发现。直到工程师偶然 ssh 到节点执行 `nvidia-smi` 才察觉，已持续空跑数周。

**场景二：宿主机裸进程导致 OOM**
某研究员直接在 GPU 节点跑实验，进程不经过 HAMI。K8s 显示 GPU 显存充足，实际上宿主机进程已占满。随后通过 K8s 调度的 vLLM Pod OOM 反复重启，花费 1.5 小时排查才定位根因。

**场景三：vLLM 参数调优耗时三天**
新模型上线前需调优 `max_num_batched_tokens` 等参数，工程师靠经验摸索，每次改参数后压测等待，三天才找到较优配置。期间推理服务部分不可用。

## 二、各子域建设方案

### 2.1 GPU 全链路可观测

**目标**：DCGM Exporter 采集全量 GPU 指标，关键异常 1 分钟内告警，AI 自动生成诊断摘要。

**部署流程**：

```
DCGM Exporter DaemonSet 部署（所有 GPU 节点）
  ↓
Prometheus ServiceMonitor 抓取 GPU 指标
  ↓
Grafana Dashboard（ID: 12239 GPU 总览 + 自定义 Pod/服务维度）
  ↓
Alertmanager 告警规则（空跑 / OOM 风险 / 温度 / 硬件错误）
  ↓
告警触发 → AI 自动生成诊断摘要 → 飞书推送
```

**关键告警规则**：

| 告警名称 | 条件 | 级别 | 处置方向 |
|---------|------|------|---------|
| GPU 空跑告警 | SM 利用率 < 5% 持续 2h | Warning | 检查宿主机裸进程，联系服务负责人 |
| OOM 风险告警 | 显存使用率 > 90% 持续 10min | Critical | 评估是否需要扩容或降低并发 |
| GPU 温度过高 | 温度 > 85°C | Warning | 检查散热，降低 GPU 负载 |
| 硬件错误告警 | ECC SBE 持续增长 | Critical | 联系运维评估更换 |

### 2.2 宿主机裸进程检测

**目标**：每小时自动扫描全部 GPU 节点，发现裸进程立即告警，消除资源黑洞。

**检测流程**：

```
K8s CronJob（每小时）
  ↓
SSH 到各 GPU 节点执行检测脚本：
  - nvidia-smi 获取所有计算进程 PID
  - 对每个 PID 检查 /proc/{pid}/cgroup 是否含 kubepods
  - 未经 K8s 调度的进程 = 裸进程
  ↓
发现裸进程 → 飞书告警（节点/PID/进程名/占用显存）
  ↓
AI Infra 工程师处置（迁移到 K8s 或确认为合规进程）
```

### 2.3 推理服务规范化（vLLM）

**目标**：所有 vLLM 实例统一由 K8s 管理，版本锁定，部署流程自动化。

**模型部署流水线**：

```
提交部署 MR（更新 Helm values：模型路径/版本/资源配额）
  ↓
GitLab CI 流水线：
  阶段1 - 上传校验（SHA256 校验 + MinIO 上传 + MLflow 版本登记）
  阶段2 - K8s 滚动更新（等待 Pod Ready + vLLM /health 健康检查）
  阶段3 - 自动验证（推送测试请求，断言响应时间 < 5s，content 非空）
  阶段4 - 路由更新（Ingress 更新，旧版本保留 24h 备回滚）
  ↓
飞书通知（成功/失败 + 耗时 + 版本信息）
失败 → 自动回滚上一版本
```

**vLLM 参数 AI 辅助调优流程**：

```
输入：模型名称 + GPU 型号 + 硬件配置 + 并发需求
  ↓
Claude API 给出推荐参数（tensor_parallel_size / max_num_batched_tokens 等）
  ↓
Locust 压测验证（QPS / P50 / P99 指标）
  ↓
AI 分析压测结果，给出调整建议
  ↓
记录到调优知识库（模型 + 硬件 + 参数 + 效果，供下次参考）
```

### 2.4 AI 模型治理

**目标**：模型资产版本化管理，实验可追溯，生产模型可快速回滚。

**MinIO 模型存储规范**：

```
model-registry/
└── {model_family}/          # qwen / llama / embedding
    └── {model_name}/
        └── {version}/       # v1.0.0 / v2.0.0
            ├── model.safetensors
            ├── config.json
            └── sha256.txt   # 完整性校验文件
```

**MLflow 记录内容**：
- 模型来源（HuggingFace ID / 微调产出 / 外部供应商）
- 显存需求 / 量化方式（FP16 / AWQ / GPTQ）
- 性能基准（Locust 压测结果：QPS / P99 延迟）
- 生产部署记录（哪个 Pipeline 部署到哪个环境）

## 五、建设路径

### Phase 0（第 1-2 周）：可观测补盲

| 任务 | 具体行动 | 验收标准 | Owner |
|------|---------|---------|-------|
| DCGM Exporter 部署 | helm install 到 monitoring namespace，接入 Prometheus ServiceMonitor | Grafana 可查 GPU SM/显存/温度指标 | AI Infra |
| Grafana GPU Dashboard | 导入 Dashboard ID 12239，补充 Pod 维度 Panel | 可按 Pod/服务查看 GPU 占用 | AI Infra |
| 宿主机裸进程检测 | 部署 CronJob + 飞书告警 | 发现裸进程 1h 内告警 | AI Infra |
| 关键告警规则 | 空跑 / OOM 风险 / 温度 3 类规则上线 | 告警测试触发正确，飞书可收到 | AI Infra |

### Phase 1（第 3-4 周）：调优与部署效率

| 任务 | 具体行动 | 验收标准 | Owner |
|------|---------|---------|-------|
| vLLM K8s 化 | 将宿主机直跑的 vLLM 实例全部迁到 K8s Deployment | K8s 管理率 > 90% | AI Infra |
| Locust 压测框架 | 压测脚本覆盖 chat/embedding 核心接口 | 可输出 QPS / P50 / P99 指标 | AI Infra |

### Phase 2（第 5-8 周）：流水线与模型治理

| 任务 | 具体行动 | 验收标准 | Owner |

## 六、关键指标

| 指标 | 当前值 | Phase 2 目标 | 采集方式 |
|------|--------|-------------|---------|
| GPU 平均 SM 利用率（prod）| ~40%（估算，无精确数据）| > 60% | DCGM Exporter → Prometheus |
| GPU 监控覆盖率 | 0%（无 DCGM）| 100% | DCGM DaemonSet 节点数 / 总 GPU 节点数 |
| K8s 管理率 | ~70%（估算）| > 90% | vLLM K8s Pod 数 / 总推理进程数 |
| 模型部署时间 | 2-4 小时（手动）| < 30 分钟 | GitLab CI Pipeline 耗时 |
| 推理服务 P99 延迟 | 未采集 | < 2 秒 | Prometheus（vLLM metrics 端点）|
| 显存 OOM 事件数 | 偶发 | 0/月 | Prometheus gpu_oom_total |
| 宿主机裸进程发现时延 | 周级（靠人工）| < 1 小时 | CronJob 告警触发时间记录 |
