# AI 提效计划 - 知识管理与文档治理

> 将 AI 嵌入知识管理核心场景：自然语言知识库问答（RAG）、文档质量自动检查、文档过时提醒、经验沉淀自动化。目标：知识检索时间从 5-15 分钟减少到 < 1 分钟，文档腐烂率（过期未更新）降低 70%。

---

## 一、效率基线（现状）

| 环节 | 当前耗时（估算）| 参与人 | 主要痛点 |
|------|--------------|--------|---------|
| 查找文档/规范 | 5-15 分钟/次 | 全员 | 飞书/仓库/本地各有存放，找不到、找到了不确定是否最新 |
| 新员工 onboarding | 2-4 周自学 | 新员工 + 导师 | 知识散落，没有统一入口，导师要频繁答同类问题 |
| 文档编写（规范/Runbook）| 1-3 小时/篇 | 各模块负责人 | 从零写耗时，缺乏结构规范，质量参差 |
| 复盘/会议纪要整理 | 30-60 分钟/次 | 主持人/记录人 | 手动整理，遗漏 Action Items |
| 文档更新检查 | 季度偶发 | 无固定 Owner | 大量文档更新滞后，实际已失效但仍被参考 |

## 三、高价值机会点详细方案

### 机会1：企业知识库 AI 问答（核心场景）

**当前状态**：工程师有问题时，要在飞书文档/本仓库/头脑中搜索，找不到就直接发消息问同事，大量重复性问题消耗资深工程师时间。  
**目标状态**：一个飞书 Bot，用自然语言提问，AI 直接从知识库中检索并回答，附上原文链接。

**方案设计**：
```
Dify RAG 知识库架构：

知识库来源（优先级排序）：
  1. 本仓库所有 Markdown 文档（本 smartvision2 知识库）
  2. 核心 Runbook（SRE/运维 SOP）
  3. 常见 FAQ（各领域收集的高频问题）
  4. 架构设计文档（ADR/技术选型记录）

Dify 配置：
  - 向量化引擎：text-embedding-3-small（OpenAI）或本地模型
  - 检索方式：混合检索（语义 + 全文）
  - 回答模型：Claude claude-3-5-haiku（快速，成本低）
  - 引用来源：必须附上原文链接和文档标题

飞书 Bot 对话示例：
  工程师问：
  "K8s 中一个 Pod 一直 CrashLoopBackOff 怎么排查？"
  
  AI 回答：
  "根据知识库[排障 Runbook - Pod 异常诊断]：
   
   1. 查看 Pod 日志：kubectl logs <pod> --previous
   2. 查看 Pod 事件：kubectl describe pod <pod>
   3. 常见原因：
      - 镜像启动命令错误
      - 内存不足（OOMKilled）
      - 配置文件错误（如 Liveness Probe 失败）
   
   参考文档：[08-环境运维/Runbook - Pod异常处理.md]"

支持追问：
  工程师问："如果是 OOMKilled 怎么处理？"
  AI 继续从上下文回答...
```

**工具栈**：Dify（RAG）+ Claude API + 飞书 Bot  
**前置条件**：核心文档已整理（本仓库已有）；Dify 自托管环境  
**实施周期**：1-2 周（Dify 部署 + 知识库导入 + 飞书 Bot 接入）  
**ROI 估算**：每天节省全员合计 30-60 分钟查找时间；新员工 onboarding 速度提升 2 倍

### 机会2：文档质量 AI 自动检查

**当前状态**：文档质量参差不齐，有的缺乏 Owner，有的没有最后更新时间，有的内容过于简单没有实操价值，但没有系统化的质量检查。  
**目标状态**：新文档提交时 CI 自动检查质量，过期文档定期自动提醒 Owner 更新。

**方案设计**：
```
GitLab CI 文档质量检查（MR 时触发）：

检查维度：
  1. 结构完整性（必填字段检查）
     - 是否有 Owner/负责人字段
     - 是否有最后更新时间
     - 是否有适用范围/前置条件
  
  2. 内容质量（AI 评估）
     Prompt:
     "评估以下文档的质量（0-100分）：
      {doc_content}
      
      评估维度：
      - 可操作性（有具体命令/步骤 vs 空洞描述）
      - 完整性（关键信息是否遗漏）
      - 准确性（技术描述是否合理）
      - 时效性风险（是否有版本号/日期相关内容可能过时）
      
      输出：分数 + 主要问题（不超过3条）+ 改进建议"

文档过期检测（每月执行）：
  - 查找超过 6 个月未修改的文档
  - 对比 git blame 找到最近修改者
  - 自动发送飞书消息给 Owner：
    "📝 文档更新提醒：[文档名] 已 7 个月未更新，
     请确认内容是否仍然准确，或标注为 [已废弃]"
```

**工具栈**：GitLab CI + Claude API + git blame + 飞书 Bot  
**前置条件**：文档有明确的 Owner 字段（或 git 提交者）  
**实施周期**：1 周  
**ROI 估算**：文档腐烂率降低 70%；新文档质量基线提升

### 机会3：会议纪要 AI 整理

**当前状态**：技术会议、复盘会议后，纪要整理靠手动，经常遗漏 Action Items，Action Items 没有 Owner 和截止日期。  
**目标状态**：会议结束后，飞书妙记 + AI 自动生成结构化纪要，包含决策记录、Action Items（含 Owner + 截止日期）。

**方案设计**：
```
飞书妙记（自动录音+转写）→ AI 整理 → 飞书文档

AI 处理 Prompt:
"根据以下会议转写内容，生成结构化会议纪要：

{transcript}

输出格式：
# 会议纪要 - {会议主题} - {日期}

## 参会人员
（从转写中识别发言者）

## 主要讨论内容
（按议题分段，每段 2-4 句话总结，不要逐字逐句）

## 关键决策
（明确做出了哪些决定，各1行）

## Action Items
| 事项 | Owner | 截止日期 | 优先级 |
|------|-------|---------|------|
| ... | ... | ... | ... |

## 待确认问题
（会议中未解决、需要进一步跟进的问题）"

自动发布：
  1. 发布到飞书群/文档
  2. Action Items 逐一创建飞书任务，指派给 Owner
```

**工具栈**：飞书妙记（录音+转写）+ Claude API + 飞书文档 API + 飞书任务 API  
**前置条件**：飞书妙记已启用；有飞书企业版 API 权限  
**实施周期**：3-5 天（飞书 API 对接）  
**ROI 估算**：纪要整理从 30-60 分钟减少到 5 分钟；Action Items 遗漏率从 ~50% 降低到 ~10%

### 机会4：新员工 AI Onboarding 助手

**当前状态**：新员工入职时，需要 2-4 周时间自己摸索，占用大量导师时间解答重复性问题，新员工上手慢影响团队效率。  
**目标状态**：AI 助手充当"第一个导师"，解答 80% 的常见问题，让导师聚焦于更高价值的引导。

**方案设计**：
```
基于知识库 RAG 的 Onboarding 专项助手：

在 Dify 创建"新员工助手"应用（与通用知识库分开）：

知识库范围：
  - 新员工必读文档
  - 各系统账号申请流程
  - 开发环境搭建 SOP
  - 团队工作规范（代码规范/PR 规范/会议规范）
  - 常用工具使用教程
  - FAQ（高频问题汇总）

引导式对话设计：
  新员工首次对话：
  AI: "你好！我是 SmartVision 技术助手，专门帮助新同学快速上手。
       你目前是什么角色（研发/运维/产品）？
       让我为你推荐最相关的入职材料。"
  
  后续可以问：
  "GitLab 账号怎么申请？"
  "本地开发环境怎么搭建？"
  "代码 PR 需要几个人 Review？"
  "K8s 集群的 kubeconfig 怎么获取？"

持续优化：
  - 统计高频问题 → 补充到 FAQ 文档
  - 新员工满意度反馈（👍/👎）→ 优化知识库覆盖
```

**工具栈**：Dify（专项知识库）+ 飞书 Bot  
**前置条件**：Dify 已部署（依赖机会1）；新员工文档已整理  
**实施周期**：1 周（在机会1基础上）  
**ROI 估算**：导师答疑时间减少 50%；新员工自主解决基础问题速度提升 3 倍

## 六、风险与回退

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| AI 回答错误（知识库内容过时）| 工程师按错误信息操作，引发事故 | 所有 AI 回答附上原文链接；用户可一键反馈"回答有误"；高风险操作类文档注明"执行前请确认文档时效" |
| 知识库敏感信息泄露（密码/密钥混入文档）| 通过 AI 问答暴露敏感信息 | 导入前扫描文档中的 secret 模式（gitleaks 规则）；知识库不导入包含真实凭证的文档 |
| Dify 服务不稳定 | AI 问答不可用 | AI 助手是辅助工具，不可用时回退到飞书搜索/直接翻文档 |
| 文档质量 CI 误拦截 | 正常文档无法合并 | 文档质量检查为"建议"级别，不阻塞 MR 合并；只是给出评分和改进建议 |
