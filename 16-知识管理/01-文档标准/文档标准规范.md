# 文档标准规范

> 标准化的文档模板和编写规范，确保文档一致、完整、可维护。

---

## 一、文档类型模板

### 服务 README 模板

```markdown
# {服务名称}

## 概述
{服务职责、定位、一句话说明}

## 架构
{架构图或关键组件说明}

## 依赖
| 依赖 | 版本 | 说明 |
|------|------|------|

## API
{API 文档链接或概要}

## 部署
{部署方式、环境要求、配置说明}

## 运维
{日志、监控、告警、常见问题}
```

### Runbook 模板

```markdown
# {场景名称} Runbook

## 症状
{故障表现、告警信息}

## 影响
{影响范围、严重程度}

## 排查步骤
1. {第一步}
2. {第二步}

## 恢复步骤
1. {第一步}

## 验证
{如何确认恢复}
```

---

## 二、文档编写原则

| 原则 | 说明 |
|------|------|
| **一句话开头** | 每篇文档第一句话说明"这是什么" |
| **面向读者** | 明确读者是谁，假设的知识水平 |
| **可操作** | 运维文档必须步骤清晰，可复制执行 |
| **链接而非复制** | 引用其他文档用链接，不要复制内容 |
| **标记过时** | 过时内容必须标注"已过时"并提供替代链接 |

---

## 三、文档 Review

- 重要文档（架构、API、复盘）需要至少 1 人 Review
- Review 关注：准确性、完整性、可读性
- 文档修改后更新"最后更新时间"
