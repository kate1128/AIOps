# 文档检索与导航规范

> 定义知识库的命名规范、目录导航结构和检索优化方法，让团队成员能快速找到所需文档。

---

## 一、文档命名规范

### 命名原则

- 使用中文，简洁明确，能从名称判断内容
- 避免使用日期作为文档名（用 Git 版本管理时间）
- 规范性文档统一后缀：`XXX规范.md`
- 总览性文档：`体系建设总览.md`
- 操作 SOP：`XXX操作手册.md` 或 `XXX-playbook.md`

### 命名示例

| 文档类型 | ✅ 好的命名 | ❌ 不好的命名 |
|---------|-----------|------------|
| 规范文档 | `PRD规范.md` | `prd-v2.md`、`PRD文档(最终版).md` |
| 操作手册 | `PostgreSQL备份操作手册.md` | `数据库.md`、`DB_backup_20240101.md` |
| 故障复盘 | `2025-01-15-GPU节点OOM故障复盘.md` | `故障复盘.md` |
| 会议纪要 | `2025-01-架构评审-Milvus选型.md` | `会议记录.md` |

## 三、文档内部交叉引用规范

### 引用格式

```markdown
# 标准内部引用（相对路径）
详见 [PRD规范](../../01-产品需求管理/02-PRD规范/PRD规范.md)

# 引用特定章节
详见 [SLO 设计](../../10-SRE%20稳定性工程/01-SLO与错误预算/SLO规范.md#一slo-定义)

# 引用另一个领域的 FAQ
常见问题参考 [中间件 FAQ](../../05-中间件/FAQ.md)
```

### 引用注意事项

- 路径区分大小写（Windows 不区分，但 Linux CI 区分）
- 含空格的目录名使用 `%20` 编码：`10-SRE%20稳定性工程`
- 避免绝对路径引用（导致在其他机器上失效）

## 四、飞书知识库搜索优化

如果团队使用飞书文档同步本知识库，以下方法提升搜索效率：

### 关键词标签规范

在文档开头（标题下方）添加关键词标签，提升搜索命中率：

```markdown
# PostgreSQL 备份操作手册

> **关键词**：PostgreSQL、数据库、备份、恢复、pg_dump、WAL、RTO、RPO

...（正文内容）
```

### 文档摘要规范

每篇规范文档的第一段（`>` 引用块）作为摘要，应：
- 一句话说清文档的用途
- 包含关键业务场景词
- 不超过 50 字

## 五、RAG 知识库接入规范

当接入 AI 知识库（如基于 RAG 的内部搜索）时，优化文档以提升召回质量：

### 文档切分友好写法

```markdown
# ✅ 好的写法（每个章节独立成段，便于切片）

## 三、备份验证频率

生产环境的 PostgreSQL 备份应按以下频率进行验证：

- **每周**：使用备份文件在测试环境恢复，验证数据完整性
- **每月**：完整恢复演练，包含业务数据校验
- **每季度**：灾难恢复演练，模拟主库不可用场景

验证命令：
```bash
pg_restore -d testdb backup.dump
psql -d testdb -c "SELECT COUNT(*) FROM users;"
```

---

# ❌ 避免的写法（内容混杂，不易切分）

多种类型内容塞在同一段落，不换行，无结构...
```

### Embedding 优化技巧

- 表格内容在 RAG 中效果差，重要表格内容同时提供文字描述版本
- 代码块单独成节（有明确标题），不要让代码和说明混杂
- 避免使用大量缩写，或在首次使用时标注全称（如 `SLO（服务级别目标）`）
