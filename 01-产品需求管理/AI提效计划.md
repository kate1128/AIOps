# AI 提效计划 - 产品需求管理

> 将 AI 嵌入需求全生命周期：从需求分类、PRD 起草、评审准备到追踪预警，目标：单需求从收集到评审通过的周期缩短 30%，PRD 撰写人力投入减少 50%。

---

## 一、效率基线（现状）

| 环节 | 当前耗时（估算）| 参与人 | 主要痛点 |
|------|--------------|--------|---------|
| 需求原始录入与分类 | 2-3 小时/周 | 产品经理 | 来自多渠道的需求手动汇总，分类标准不统一 |
| PRD 初稿撰写 | 1-2 人天/需求 | 产品经理 | 背景、用户故事、验收标准需从零起草 |
| 需求评审材料准备 | 4-6 小时/次 | 产品经理 | 汇总背景资料、竞品参考、技术前置，工作量大 |
| 评审会议纪要整理 | 1-2 小时/次 | 产品经理 | 手动整理决议和 Action Items |
| 需求优先级排序 | 0.5-1 天/迭代 | 产品负责人 | RICE 评分主观，缺少数据支撑 |

## 二、AI 工具全景

| 工具/能力 | 适用场景 | 使用状态 | 接入难度 | 月费用参考 |
|----------|---------|---------|---------|----------|

## 三、高价值机会点详细方案

### 机会1：多渠道需求自动汇聚与分类

**当前状态**：需求来自销售反馈（飞书）、客户群（微信/飞书）、用户反馈表单、内部技术债，散落各处，人工汇总漏项多。  
**目标状态**：所有需求自动流入统一需求池（Jira / 飞书多维表格），AI 自动完成分类打标。

**方案设计**：
```
需求来源 → 统一收集入口 → AI 自动分类 → 需求池

收集入口：
  - 飞书表单（内部需求提报）
  - 自动同步飞书群关键词消息（含"需求"/"建议"/"问题"）
  - 邮件转发到指定地址自动入库

AI 分类打标（Claude API）：
  字段：
  - 功能模块（智能问答/文档管理/权限/性能/其他）
  - 需求类型（新功能/优化/Bug/技术债）
  - 影响用户群（内部/特定客户/全部客户）
  - 初步优先级（P0/P1/P2）及置信度
  - 相似需求去重提示（Embedding 相似度检索）

Prompt 示例：
  "请分析以下需求描述，输出 JSON 格式：
   {module, type, affected_users, priority, confidence, summary_20chars}
   需求描述：{text}"
```

**工具栈**：n8n + Claude API + 飞书多维表格  
**前置条件**：定义好需求模块和类型的标准枚举值  
**实施周期**：5-7 天  
**ROI 估算**：节省 2 小时/周（汇聚分类），需求遗漏率降低 ~60%

### 机会2：AI 辅助 PRD 快速起草

**当前状态**：产品经理从空白 PRD 模板开始填写，背景描述、用户故事等框架性内容占用大量精力。  
**目标状态**：输入需求一句话描述 + 背景，AI 在 2 分钟内生成结构完整的 PRD 草稿，产品经理专注补充核心业务逻辑。

**方案设计**：
```
输入（产品经理提供）：
  - 功能名称
  - 解决什么问题（用户痛点，2-3句话）
  - 目标用户（从已定义的用户分层中选择）
  - 核心流程描述（流程步骤，可列点）
  - 约束条件（不做什么/依赖项）

AI 输出 PRD 草稿：
  ## 背景与目标
  ## 用户故事（格式：作为...，我希望...，以便...）
  ## 功能说明
    - 核心流程（泳道图文字描述）
    - 边界情况处理
    - 错误场景
  ## 验收标准（Acceptance Criteria，Given-When-Then 格式）
  ## 数据指标（建议埋点事件清单）
  ## 技术依赖（需提前识别的 API/数据/基础设施依赖）
  ## 风险与假设

知识库增强：
  - 历史 PRD 作为 Few-Shot 示例（取最近 10 份质量较好的）
  - 产品设计原则文档作为约束上下文
```

**工具选型**：  
- 快速验证：Claude Projects（上传历史PRD作为上下文，1天内可用）  
- 正式方案：Dify 工作流（接入内部知识库，更稳定可控）  

**前置条件**：整理 5-10 份历史高质量 PRD 作为示例  
**实施周期**：快速验证 1 天；Dify 正式版 1-2 周  
**ROI 估算**：PRD 初稿时间从 1-2 人天缩短到 2-4 小时，降低约 60%

### 机会3：需求评审材料 AI 自动准备

**当前状态**：评审前产品经理需手动整理：背景说明、竞品现状、技术前置确认、历史相关需求。通常需要 4-6 小时。  
**目标状态**：点击"生成评审包"，AI 在 10 分钟内自动组装评审所需材料。

**方案设计**：
```
输入：PRD 文档 + 评审类型（需求评审/技术评审/设计评审）

AI 自动组装评审包：
  1. 背景摘要（从 PRD 提炼 5 要点）
  2. 竞品现状（触发竞品监控系统查询相关功能）
  3. 关联历史需求（Embedding 搜索需求库中相似项）
  4. 技术影响面（标记可能涉及的模块，供技术同学提前评估）
  5. 评审议程建议（根据 PRD 复杂度生成议程）
  6. 待确认问题清单（AI 预判可能的争议点）
```

**工具栈**：Dify 工作流 + Claude + 竞品知识库  
**前置条件**：竞品监控 (机会1 中的工作流) 已运行 ≥ 1 个月有数据  
**实施周期**：2-3 周（依赖竞品知识库）  
**ROI 估算**：评审准备时间从 4-6 小时减少到 30 分钟

### 机会4：评审会议纪要 AI 自动生成

**当前状态**：评审会议后需人工整理决议和 Action Items，通常拖延到第二天，遗漏率高。  
**目标状态**：会议结束后 5 分钟内，飞书妙记自动生成纪要草稿，含结构化 Action Items。

**方案设计**：
```
飞书妙记自动转录 → 结构化纪要生成 → Action Items 自动创建 Jira 任务

纪要结构：
  - 核心决策（3-5 条）
  - 待确认事项（含负责人和截止日期）
  - Action Items（自动创建 Jira 子任务）
  - 下次评审触发条件

Prompt：
  "根据以下会议转录，提取：
   1. 最终决策（已达成共识的）
   2. 待确认事项（有分歧或需要进一步确认的）
   3. Action Items（格式：[负责人][截止日期] 任务描述）
   转录内容：{transcript}"
```

**工具栈**：飞书妙记（已有）+ Claude API + Jira API  
**前置条件**：评审会议在飞书视频会议中进行（支持录制/转录）  
**实施周期**：3-5 天  
**ROI 估算**：纪要整理从 1-2 小时缩短到 10 分钟确认，Action Items 遗漏率接近零

## 四、实施路径

### Phase 0（第 1 周）：低成本验证

| 任务 | 具体行动 | 验收标准 | Owner |
|------|---------|---------|-------|
| PRD 草稿 AI 验证 | 在 Claude Projects 上传 5 份历史 PRD，起草 2 个真实需求 | 产品经理评分：节省时间 > 50%，草稿质量 ≥ 7/10 | 产品经理 |
| 飞书妙记纪要试用 | 在下次评审会议中开启妙记录制，测试自动摘要效果 | 纪要覆盖度 ≥ 80% 的关键决策 | 产品经理 |
| 需求分类 Prompt 设计 | 为需求分类场景编写并测试 Prompt，用最近 50 条需求验证 | 分类准确率 > 85%（人工抽检 20 条）| 产品经理 |

### Phase 1（第 2-3 周）：核心流程自动化

| 任务 | 具体行动 | 验收标准 | Owner | 前置条件 |
|------|---------|---------|-------|---------|
| 需求汇聚工作流 | n8n 接入飞书表单 + AI 分类，结果写入需求池 | 新需求提交后 5 分钟内完成分类 | 运维/产品 | 飞书表单已建立 |
| 纪要自动化接入 | 飞书妙记 → Claude API → 结构化纪要模板 → 自动发送到频道 | 每次评审会后自动产出纪要草稿 | 运维 | 飞书妙记权限 |
| PRD 助手 Dify 化 | 在 Dify 部署 PRD 助手 WorkFlow，替代 Claude Projects | 5 人以上产品团队均可使用，响应 < 15s | 运维 | Dify 内网部署 |

### Phase 2（第 4-6 周）：质量提升与闭环

| 任务 | 具体行动 | 验收标准 | Owner | 前置条件 |
|------|---------|---------|-------|---------|
| 评审材料自动组装 | Dify 工作流接入竞品知识库，实现评审包一键生成 | 评审准备时间 < 30 分钟 | 产品 + 运维 | 竞品监控有 ≥ 1 个月数据 |
| Jira Action Items 自动创建 | 纪要中的 Action Items 自动同步到 Jira | 纪要 100% 触发 Jira 任务创建 | 运维 | Jira API 权限 |
| AI 提效数据回顾 | 统计 PRD 撰写时间、评审准备时间变化 | 人力投入对比基线减少 ≥ 40% | 产品负责人 | Phase 1 完成 |

## 六、风险与回退

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| PRD 草稿偏离业务上下文 | 产品经理大量修改，提效失效 | 建立草稿质量评分（1-10），低于 6 分则报警并优化 Prompt |
| 需求分类错误影响优先级判断 | 高优需求被降级 | 分类结果标记置信度，低于 80% 的人工复核 |
| 飞书妙记转录质量差（方言/专业术语）| 纪要覆盖不全 | 提前准备术语词典；重要评审结论人工核对一遍 |
| Action Items 自动创建 Jira 权限问题 | 集成失效 | 回退到手动复制纪要到 Jira |
