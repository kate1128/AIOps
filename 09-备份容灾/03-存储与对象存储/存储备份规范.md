# 存储备份规范

## 1. 目标

保护对象存储（MinIO）和持久化卷（PVC）中的数据：

- MinIO bucket 数据异地/多副本保护
- K8s PVC 数据可快照、可恢复
- RPO 小于等于 24 小时，RTO 小于等于 8 小时

## 2. MinIO 备份方案

### 2.1 Bucket 复制（主要方案）

开启 MinIO 站点复制或 bucket 级别复制，将数据实时同步到备用实例：

```bash
# 配置 MinIO Client
mc alias set primary http://minio-primary:9000 ACCESS_KEY SECRET_KEY
mc alias set backup  http://minio-backup:9000  ACCESS_KEY SECRET_KEY

# 开启 bucket 复制
mc replicate add primary/production-bucket \
  --remote-bucket backup/production-bucket-replica \
  --priority 1
```

**站点级别双向复制**（推荐）：

```bash
mc admin replicate add primary backup
```

### 2.2 定期全量同步（兜底方案）

```bash
# 每日全量同步到归档桶（增量模式）
mc mirror --overwrite primary/production-bucket backup/production-bucket-archive
```

### 2.3 备份保留策略

```bash
# 开启版本控制（防止误删）
mc version enable primary/production-bucket

# 配置生命周期（旧版本 30 天后移至冷存储）
mc ilm add primary/production-bucket \
  --noncurrent-days 30 \
  --storage-class GLACIER
```

## 3. K8s PVC 备份（Velero）

### 3.1 安装 Velero

```bash
velero install \
  --provider aws \
  --plugins velero/velero-plugin-for-aws:v1.8.0 \
  --bucket velero-backup \
  --backup-location-config region=minio,s3ForcePathStyle=true,s3Url=http://minio:9000 \
  --use-volume-snapshots=false \
  --secret-file ./minio-credentials
```

### 3.2 定期 PVC 备份

```bash
# 创建每日备份计划（备份含 PVC 的命名空间）
velero schedule create daily-backup \
  --schedule="0 1 * * *" \
  --include-namespaces production,ai-infra \
  --ttl 720h
```

### 3.3 PVC 恢复流程

```text
1. 确认备份列表：velero backup get
2. 选择恢复点：velero backup describe <backup-name>
3. 执行恢复：velero restore create --from-backup <backup-name>
4. 验证 Pod 和 PVC 状态正常
5. 检查数据完整性
```

## 4. 不适合备份的存储

| 存储类型 | 原因 | 替代方案 |
| --- | --- | --- |
| 临时缓存（Redis 无持久化） | 数据可重建 | 不备份 |
| 日志归档（已到 Loki） | 可接受部分丢失 | 延长保留期 |
| 构建缓存 | 可重新生成 | 不备份 |

## 5. KPI

| 指标 | 目标值 |
| --- | --- |
| MinIO 复制延迟 | 小于等于 5 分钟（正常网络） |
| PVC 备份成功率 | 大于等于 99%/周 |
| 存储恢复演练 RTO | 小于等于 8 小时（季度演练） |
| 版本控制覆盖率 | 100% 生产 bucket |
