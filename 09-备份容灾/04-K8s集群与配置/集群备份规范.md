# K8s 集群与配置备份规范

## 1. 目标

确保 K8s 集群在以下场景下可完整恢复：

- etcd 数据损坏或丢失
- 控制平面节点故障
- 误删命名空间或关键资源
- 整集群迁移或重建

RPO 小于等于 1 小时，RTO 小于等于 2 小时（配置恢复），业务恢复依赖应用自身。

## 2. 备份层次

```text
K8s 集群恢复能力 = etcd 快照 + 配置 Git 化 + 镜像可用
                  ↓           ↓              ↓
               集群状态    应用配置声明    镜像仓库 Harbor
```

三者缺一不可，缺 etcd 快照无法恢复集群状态，缺 Git 配置无法重建应用。

## 3. etcd 备份

### 3.1 自动化快照

```bash
# 生成 etcd 快照
ETCDCTL_API=3 etcdctl snapshot save /backup/etcd-$(date +%Y%m%d-%H%M).db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key

# 验证快照
ETCDCTL_API=3 etcdctl snapshot status /backup/etcd-*.db --write-out=table

# 上传到 MinIO
mc cp /backup/etcd-*.db minio/backup-k8s/etcd/
```

K8s CronJob 示例（每 6 小时执行）：

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: etcd-backup
  namespace: kube-system
spec:
  schedule: "0 */6 * * *"
  jobTemplate:
    spec:
      template:
        spec:
          hostNetwork: true
          containers:
            - name: etcd-backup
              image: registry.k8s.io/etcd:3.5.9-0
              command: ["/bin/sh", "/scripts/etcd-backup.sh"]
              volumeMounts:
                - name: etcd-certs
                  mountPath: /etc/kubernetes/pki/etcd
          volumes:
            - name: etcd-certs
              hostPath:
                path: /etc/kubernetes/pki/etcd
          restartPolicy: OnFailure
          nodeSelector:
            node-role.kubernetes.io/control-plane: ""
```

### 3.2 etcd 恢复流程

```text
1. 停止所有控制平面组件（API Server、Controller Manager、Scheduler）
2. 从 MinIO 下载最近 etcd 快照
3. 恢复快照到所有 etcd 节点
   etcdctl snapshot restore etcd-backup.db \
     --name <node-name> \
     --initial-cluster <cluster-config> \
     --data-dir /var/lib/etcd-restore
4. 更新 etcd static pod manifest 指向新数据目录
5. 重启控制平面组件
6. 验证集群状态：kubectl get nodes
```

## 4. 配置 Git 化（GitOps）

所有 K8s 资源声明必须在 Git 中存在：

| 配置类型 | Git 存储方式 | 示例 |
| --- | --- | --- |
| Helm values | 仓库 + 目录分环境 | `helm/production/values.yaml` |
| Kustomize overlay | 仓库 + 目录结构 | `overlays/production/` |
| ArgoCD Application | 声明式配置入 Git | `argocd/apps/` |
| 命名空间/RBAC | 基础设施仓库 | `infra/namespaces/` |

**不允许**通过 `kubectl apply` 直接修改生产资源而不同步到 Git。

## 5. 镜像可用性保障

- Harbor 使用持久化存储（PVC），且 PVC 已纳入 Velero 备份
- 关键基础镜像在 Harbor 本地存有副本（不依赖 docker.io 实时拉取）
- Harbor 配置（用户、项目、规则）定期导出备份

## 6. KPI

| 指标 | 目标值 |
| --- | --- |
| etcd 快照备份成功率 | 大于等于 99.9%（每 6 小时） |
| etcd 快照保留数量 | 最近 30 天 |
| 应用配置 Git 化覆盖率 | 100% 生产应用 |
| 集群恢复演练 RTO | 小于等于 2 小时（季度演练） |
