# 数据库备份规范

## 1. 目标

确保生产数据库数据可恢复，满足：

- RPO 小于等于 1 小时（最大数据丢失 1 小时以内）
- RTO 小于等于 4 小时（4 小时内恢复可用）
- 备份可验证，恢复流程文档化

## 2. 支持的数据库类型

| 数据库 | 全量备份方式 | 增量/实时备份 | 时间点恢复 |
| --- | --- | --- | --- |
| MySQL | mysqldump / Xtrabackup | binlog | 支持（基于 binlog） |
| PostgreSQL | pg_dump / pg_basebackup | WAL 归档 | 支持（基于 WAL） |
| Redis | RDB 快照 / AOF | AOF 实时 | 有限支持 |

## 3. 执行流程

### 3.1 MySQL 自动化备份

```bash
# 全量备份（每日凌晨 2 点执行）
mysqldump -h $DB_HOST -u $DB_USER -p$DB_PASS \
  --all-databases \
  --single-transaction \
  --flush-logs \
  --master-data=2 \
  | gzip > backup-$(date +%Y%m%d).sql.gz

# 推送到 MinIO
mc cp backup-$(date +%Y%m%d).sql.gz minio/backup-db/mysql/
```

建议使用 K8s CronJob 调度：

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: mysql-backup
  namespace: backup
spec:
  schedule: "0 2 * * *"
  jobTemplate:
    spec:
      template:
        spec:
          containers:
            - name: backup
              image: mysql:8.0
              command: ["/bin/sh", "/scripts/backup.sh"]
              envFrom:
                - secretRef:
                    name: mysql-backup-credentials
          restartPolicy: OnFailure
```

### 3.2 PostgreSQL 自动化备份

```bash
# 全量备份
pg_basebackup -h $PG_HOST -U $PG_USER -D /backup/pg-$(date +%Y%m%d) \
  --format=tar --compress=9 --wal-method=stream

# 推送到 MinIO
mc cp /backup/pg-$(date +%Y%m%d).tar.gz minio/backup-db/postgresql/
```

WAL 归档配置（`postgresql.conf`）：

```text
archive_mode = on
archive_command = 'mc cp %p minio/backup-db/postgresql/wal/%f'
wal_level = replica
```

### 3.3 恢复流程

**MySQL 时间点恢复步骤**：

```text
1. 从 MinIO 下载最近一次全量备份
2. 恢复全量备份到目标实例
   mysql < backup-YYYYMMDD.sql
3. 应用 binlog 到目标时间点
   mysqlbinlog --stop-datetime="2026-05-29 10:00:00" binlog.* | mysql
4. 验证数据完整性（行数、关键表记录）
5. 切换应用连接到恢复实例
```

**PostgreSQL 时间点恢复（PITR）步骤**：

```text
1. 下载 base backup
2. 配置 recovery.conf（指定目标时间）
   recovery_target_time = '2026-05-29 10:00:00'
3. 从 MinIO 拉取 WAL 文件
4. 启动 PostgreSQL，等待 recovery 完成
5. 执行 SELECT pg_wal_replay_resume() 解除恢复模式
```

## 4. 备份验证

每季度执行一次恢复演练：

- [ ] 从备份文件完整恢复到测试实例
- [ ] 验证核心表数据行数与生产一致
- [ ] 记录实测 RTO，与目标对比
- [ ] 输出演练报告（见 06-容灾演练）

## 5. KPI

| 指标 | 目标值 |
| --- | --- |
| 全量备份成功率 | 大于等于 99.9%/月 |
| 备份文件上传 MinIO 成功率 | 大于等于 99.9%/月 |
| 恢复演练 RTO 达标率 | 100%（季度） |
| binlog/WAL 归档延迟 | 小于等于 5 分钟 |
