# 镜像与供应链安全规范

## 1. 目标

确保发布到生产的容器镜像：

- 不包含高危 CVE 漏洞
- 不残留密钥、证书等敏感文件
- 来源可验证，构建过程可追溯
- 使用最小化基础镜像，减少攻击面

## 2. 控制项

| ID | 控制项 | 严重度 | 执行阶段 |
| --- | --- | --- | --- |
| C-01 | 镜像构建后必须执行 Trivy 漏洞扫描 | 高 | CI |
| C-02 | 存在 CRITICAL 漏洞的镜像禁止推送到生产 Registry | 高 | CI/CD |
| C-03 | 镜像中禁止残留 .env 文件、私钥或明文凭据 | 高 | CI |
| C-04 | 基础镜像使用 distroless 或 alpine，不使用 latest tag | 中 | CR |
| C-05 | 生产镜像须使用 Cosign 签名，Harbor 开启签名验证 | 中 | CI/CD |
| C-06 | SBOM 随镜像归档，存入 Harbor 或 MinIO | 低 | CI |

## 3. 执行流程

### 3.1 镜像扫描流程

```text
docker build
  ↓
CI: image-scan job
  ↓
trivy image --exit-code 1 --severity CRITICAL,HIGH <image>
  ↓
CRITICAL 存在 → 阻断推送
HIGH 存在 → 可配置为警告（建 ticket）
  ↓
trivy image --format cyclonedx → 生成 SBOM
  ↓
cosign sign <image>
  ↓
docker push to Harbor
```

### 3.2 GitLab CI 配置示例

```yaml
image-scan:
  stage: scan
  image: aquasec/trivy:latest
  script:
    - trivy image --exit-code 1 --severity CRITICAL $IMAGE_TAG
    - trivy image --format cyclonedx --output sbom.json $IMAGE_TAG
  artifacts:
    paths:
      - sbom.json

image-sign:
  stage: publish
  needs: [image-scan]
  script:
    - cosign sign --key $COSIGN_KEY $IMAGE_TAG
```

### 3.3 镜像残留密钥检测

```bash
# 检查镜像文件系统中是否含敏感文件
trivy image --scanners secret <image>
# 或导出文件系统手动检查
docker export $(docker create <image>) | tar -t | grep -E '.env|.pem|.key|id_rsa'
```

Dockerfile 规范：

- 多阶段构建，最终阶段不复制源码和中间产物
- 不在 `RUN` 指令中使用明文密钥（用 `--secret` 或构建参数注入）
- `.dockerignore` 中排除 `.env`、`*.key`、`*.pem`

### 3.4 基础镜像管理

- 统一使用团队维护的基础镜像列表（存 Harbor 基础镜像项目）
- 基础镜像每月扫描一次，出现 CRITICAL 时通知所有下游镜像更新
- 禁止直接引用 `docker.io` 上的第三方镜像（通过 Harbor proxy cache 中转）

## 4. 工具推荐

| 场景 | 工具 | 备注 |
| --- | --- | --- |
| 镜像漏洞扫描 | Trivy | 首选，速度快 |
| 镜像签名 | Cosign | 与 Harbor 配套 |
| SBOM 生成 | Syft / Trivy | CycloneDX 格式 |
| Harbor 准入 | Harbor Notary | 签名验证 |
| Dockerfile lint | hadolint | CI 中集成 |

## 5. KPI

| 指标 | 目标值 |
| --- | --- |
| 镜像扫描覆盖率 | 100% 生产镜像 |
| CRITICAL CVE 镜像入生产率 | 0 |
| 镜像签名覆盖率 | 大于等于 80% 生产镜像 |
| SBOM 归档率 | 大于等于 80% 发版镜像 |

## 6. 验收标准

- [ ] CI pipeline 中 image-scan job 已启用且为阻断级别
- [ ] Harbor 中无 CRITICAL 漏洞镜像在运行（定期核查）
- [ ] Dockerfile 规范已写入 03-CICD建设 镜像构建规范
- [ ] 至少 1 个服务完成端到端镜像签名流程验证
