# 密钥与凭据安全规范

## 1. 目标

防止 API Key、Token、数据库密码等凭据以明文形式出现在：

- 前端代码、JS bundle、HTML、CSS
- Git 仓库（任何分支或提交历史）
- 容器镜像 layer
- 日志、错误响应、HTTP Headers

## 2. 控制项

| ID | 控制项 | 严重度 | 执行阶段 |
| --- | --- | --- | --- |
| C-01 | 禁止在前端代码中硬编码任何凭据 | 高 | CR + CI |
| C-02 | 禁止在 Git 提交中包含明文凭据 | 高 | pre-commit + CI |
| C-03 | 所有密钥通过环境变量或 Secret Manager 注入，不进源码 | 高 | CR |
| C-04 | CI/CD 变量中凭据必须标记为 Masked/Protected | 中 | CI配置 |
| C-05 | 前端仅允许访问网关签发的短期 Token，不持有原始 API Key | 高 | 架构评审 |
| C-06 | 凭据泄露事件 24 小时内必须轮换 | 高 | 应急响应 |

## 3. 执行流程

### 3.1 前端代码防泄露

前端项目中的规则：

- 所有密钥通过 `VITE_*` / `REACT_APP_*` 等环境变量读取，不硬编码
- 构建阶段运行 `gitleaks` 或 `truffleHog` 扫描 dist 产物
- Code Review checklist 中包含"是否有凭据硬编码"检查项

```text
前端构建流程
  ↓
1. 源码 secret 扫描（gitleaks）
  ↓
2. 构建 JS bundle
  ↓
3. bundle 内容扫描（grep AKIA / sk- / Bearer 等 pattern）
  ↓
4. 通过 → 发布; 失败 → 阻断 + 通知
```

### 3.2 Git pre-commit 拦截

开发者本地安装 `pre-commit` hook：

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.4
    hooks:
      - id: gitleaks
```

CI 阶段同步执行，不依赖开发者本地配置。

### 3.3 CI 阶段扫描

GitLab CI 中加入 secret detection job（与 03-CICD建设 联动）：

```yaml
secret-detection:
  stage: scan
  image: zricethezav/gitleaks:latest
  script:
    - gitleaks detect --source . --exit-code 1
  allow_failure: false
```

### 3.4 泄露处置

一旦检测到凭据泄露：

1. 立即在凭据管理平台（如 Vault / GitLab CI Var）轮换该凭据
2. 撤销旧凭据
3. 检查是否已被外部利用（查日志）
4. 若已入 Git 历史，执行 `git filter-repo` 清除
5. 填写泄露事件记录（见模板）

## 4. 工具推荐

| 场景 | 工具 | 备注 |
| --- | --- | --- |
| Git 历史扫描 | gitleaks | 开源，速度快 |
| CI 集成扫描 | GitLab Secret Detection | 内置，无需额外部署 |
| 提交前拦截 | pre-commit + gitleaks | 开发者本地 |
| 凭据存储 | GitLab CI Variables (masked) / Vault | 按团队规模选 |
| 前端 bundle 扫描 | 自定义 grep 脚本 / semgrep | 匹配常见 key pattern |

## 5. KPI

| 指标 | 目标值 |
| --- | --- |
| CI 中 secret 扫描覆盖率 | 100% 主要仓库 |
| 明文凭据入库事件 | 0 件/月 |
| 泄露后轮换响应时间 | 小于 24 小时 |
| pre-commit hook 安装率 | 大于等于 80% 活跃开发者 |

## 6. 验收标准

- [ ] 所有核心仓库 CI 中已启用 secret 扫描，`allow_failure: false`
- [ ] 前端项目无硬编码凭据（gitleaks 扫描结果清洁）
- [ ] 开发者手册包含"不要硬编码密钥"操作规范
- [ ] 已有凭据泄露应急处置流程并演练一次
