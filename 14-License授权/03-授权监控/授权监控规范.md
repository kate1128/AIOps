# 授权监控规范

> 持续监控 License 的有效性、使用量和异常行为，保障授权合规并提前预警到期风险。

---

## 一、监控目标

| 监控维度 | 目的 |
|---------|------|
| **有效性监控** | 实时感知 License 是否有效（未篡改、未过期、签名验证通过）|
| **使用量监控** | 追踪实际使用量与授权量的比例（防止超量使用）|
| **到期预警** | 提前通知相关人员续费，避免意外到期影响客户 |
| **异常行为检测** | 发现 License 被复制到未授权环境等安全风险 |

---

## 二、License 验证日志规范

应用在每次验证 License 时，必须输出结构化日志：

```json
{
  "timestamp": "2025-01-15T10:00:00Z",
  "event": "license_verify",
  "client_id": "customer-abc",
  "license_key": "LIC-xxxxx",
  "result": "valid",           // valid | expired | invalid_sig | over_quota
  "details": {
    "valid_until": "2026-01-15",
    "days_remaining": 365,
    "features": ["core", "ai", "premium"],
    "quota_used": 450,
    "quota_total": 500,
    "machine_id": "xxxxxxxx"   // 硬件指纹，用于检测跨机器使用
  }
}
```

### 日志采集

License 验证日志统一输出到应用 stdout，通过 Loki 采集：

```yaml
# 在 Grafana Loki 中配置 License 日志 alert
# logql: 筛选 license 验证失败日志
{app="wenxue-backend"} |= "license_verify" | json | result != "valid"
```

## 三、使用量指标采集

应用通过 Prometheus metrics endpoint 暴露 License 使用量：

```go
// 示例：Prometheus 指标定义
var (
  licenseQuotaTotal = prometheus.NewGaugeVec(
    prometheus.GaugeOpts{
      Name: "license_quota_total",
      Help: "Total license quota",
    }, []string{"client_id", "feature"},
  )
  licenseQuotaUsed = prometheus.NewGaugeVec(
    prometheus.GaugeOpts{
      Name: "license_quota_used",
      Help: "Used license quota",
    }, []string{"client_id", "feature"},
  )
  licenseDaysRemaining = prometheus.NewGaugeVec(
    prometheus.GaugeOpts{
      Name: "license_days_remaining",
      Help: "Days until license expiry",
    }, []string{"client_id"},
  )
)
```

### Grafana 看板面板（关键指标）

| 面板名称 | PromQL | 用途 |
|---------|--------|------|
| 到期天数（最小值）| `min(license_days_remaining)` | 快速发现即将到期的客户 |
| 配额使用率 | `license_quota_used / license_quota_total * 100` | 发现超量风险 |
| 验证成功率 | `rate(license_verify_total{result="valid"}[5m])` | 发现验证异常 |

## 四、到期预警机制

### 预警时间节点

| 剩余天数 | 告警接收人 | 告警方式 | 消息内容 |
|---------|---------|---------|---------|
| 60 天 | CSM（客户成功经理）| 邮件 + 飞书 | 提醒跟进续费洽谈 |
| 30 天 | CSM + 运营负责人 | 飞书 + 工单自动创建 | 确认续费进展 |
| 7 天 | CSM + 运营 + CTO | 飞书 @所有人 + 电话 | 紧急处理，防止到期影响客户 |
| 到期当天 | 全团队 | PagerDuty/电话 | 立即响应 |

### Prometheus 告警规则

```yaml
# prometheus-rules/license.yaml
groups:
  - name: license_expiry
    rules:
      - alert: LicenseExpirySoon60Days
        expr: license_days_remaining < 60
        for: 1h
        labels:
          severity: warning
        annotations:
          summary: "客户 {{ $labels.client_id }} License 将在 {{ $value }} 天后到期"

      - alert: LicenseExpirySoon7Days
        expr: license_days_remaining < 7
        for: 10m
        labels:
          severity: critical
        annotations:
          summary: "⚠️ 紧急：客户 {{ $labels.client_id }} License 仅剩 {{ $value }} 天！"

      - alert: LicenseQuotaHigh
        expr: license_quota_used / license_quota_total > 0.9
        for: 30m
        labels:
          severity: warning
        annotations:
          summary: "客户 {{ $labels.client_id }} 配额使用率超过 90%（当前 {{ $value | humanizePercentage }}）"
```

## 五、异常行为检测

### 检测规则

| 异常类型 | 检测方法 | 处理方式 |
|---------|---------|---------|
| **跨机器使用** | machine_id 与授权时不一致 | 记录告警日志，返回验证失败 |
| **签名验证失败** | ECDSA 签名校验不通过 | 立即拒绝，告警通知安全团队 |
| **License 超量** | quota_used > quota_total | 记录超量日志，降级或拒绝新请求 |
| **并发多机使用** | 同一 License 在多个不同 machine_id 上同时验证 | 疑似 License 泄露，通知商务 |

### 检测 SQL（如 License 数据入库）

```sql
-- 检测同一 License 在不同机器使用（过去 24 小时）
SELECT license_key, COUNT(DISTINCT machine_id) as machine_count
FROM license_verify_log
WHERE timestamp > NOW() - INTERVAL '24 hours'
  AND result = 'valid'
GROUP BY license_key
HAVING COUNT(DISTINCT machine_id) > 1;
```
