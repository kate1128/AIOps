# License 开通续期手册

> 运营/销售侧操作指南。覆盖 License 创建、下发、续期、升版、吊销全流程。

---

## 一、开通新 License

**适用场景**：新客户签约后，首次生成 License。

### 操作步骤

1. **登录运营后台** → 菜单「License 管理」→「新建 License」

2. **填写客户信息**

   | 字段 | 填写说明 |
   |---|---|
   | 客户 ID | 从 CRM 系统复制（格式：`customer-xxx`）|
   | 客户名称 | 公司全称，与合同一致 |
   | 版本 | 按合同选择：基础版 / 专业版 / 企业版 |
   | 有效期 | 合同到期日；永久授权填「永久」|
   | 用户数上限 | 按合同填写，不限填 `-1` |
   | GPU 节点数（AI 版）| 按合同填写 |
   | 备注 | 填写合同编号，便于后续审计 |

3. **点击「生成 License」**，系统自动签名并输出 `.lic` 文件

4. **下发给客户**（见下方「License 文件下发」）

5. **在 CRM/合同系统记录**：license_id、有效期、下发日期

---

## 二、License 文件下发

**下发方式（按安全级别排序）**：

| 方式 | 适用场景 | 操作 |
|---|---|---|
| FTP | 首选，客户可自助下载 | 上传到 FTP `/licenses/{customer_id}/` 目录，通知客户下载 |
| 加密邮件 | FTP 不可用时备选 | 发送给客户指定联系人，邮件主题：`[问学授权文件] {customer_name} v{version}` |
| 客户自助门户 | 规划中 | 客户登录门户自行下载 |

**禁止**通过飞书/微信等 IM 发送 .lic 文件（无审计记录）。

### 客户安装说明（随文件附上）

```bash
# 1. 将 license.lic 放到指定目录
cp license.lic /data/wenxue/config/

# 2. 重启服务使 License 生效（首次安装需重启）
docker restart wenxue-app

# 3. 验证 License 已生效：登录后台，查看「系统信息」页面是否显示授权信息
```

## 三、续期

**触发时机**：系统在到期前 30 天自动发邮件给运营，提醒跟进续费。

### 操作步骤

1. 确认客户已续签合同（收到财务确认或合同扫描件）

2. 运营后台 → 「License 管理」→ 找到该客户 → 「续期」

3. 填写新的到期日期（从**合同续签日**算起，非旧 License 到期日）

4. 系统生成新 `.lic` 文件（新的 `expires_at`，重新签名）

5. 按「License 文件下发」流程发给客户

6. 客户替换 `.lic` 文件后**无需重启**（热加载，5 分钟内生效）

---

## 四、版本升级（如基础版升级专业版）

1. 确认升级合同已签署

2. 运营后台 → 「License 管理」→ 找到该客户 → 「编辑」

3. 修改「版本」字段，系统自动更新 `features` 列表

4. 如有用量变化（如增加 GPU 节点数），同步修改 `limits`

5. 生成新 `.lic` 文件，发给客户

6. 客户替换 `.lic` 文件后，界面自动解锁新功能

---

## 五、吊销 License

**适用场景**：客户违约、退款、安全事件。

⚠️ **吊销是不可逆操作，执行前需经过主管审批。**

1. 运营后台 → 「License 管理」→ 找到该客户 → 「吊销」→ 填写吊销原因

2. 系统将该 `license_id` 加入吊销名单（产品下次联网校验时生效）

3. **对于完全离线客户**：需发布包含新公钥的产品更新，并通知客户升级（旧 License 在新版本中失效）

4. 在 CRM 系统更新客户状态为「已终止」

---

## 六、常见操作记录规范

每次操作后在 CRM/合同系统记录以下信息（便于审计）：

| 操作 | 必须记录 |
|---|---|
| 开通 | license_id、客户名、有效期、操作人、操作时间 |
| 续期 | 旧 license_id、新 license_id、新有效期、对应合同编号 |
| 升版 | 变更前版本、变更后版本、对应合同编号 |
| 吊销 | 吊销原因、审批人、操作时间 |
