# License 授权 FAQ

---

**Q1：私有化部署客户断网时 License 还能验证吗？**

可以。本方案使用本地签名验证（非对称加密），产品启动时读取 .lic 文件，用内嵌公钥验证签名，不需要回调服务器。客户完全离线也能正常使用，只要 .lic 文件有效且未到期。

---

**Q2：私钥泄漏了怎么办？**

这是最严重的安全事件，立即执行：
1. 生成新的 ECDSA 密钥对
2. 用新私钥为所有**有效**的 License 重新签名
3. 通知所有客户升级到包含新公钥的产品版本，并下发新 .lic 文件
4. 旧密钥签发的 License 在新版本中失效

这是私钥必须严格保管的原因：泄漏后需要通知所有客户，代价极高。

---

**Q3：客户能伪造或篡改 .lic 文件吗？**

不能。.lic 文件包含 ECDSA 数字签名，任何对内容的篡改都会导致签名验证失败，产品拒绝启动。

但需要注意：如果客户把公钥从产品中替换掉（逆向工程），理论上可以用自己的私钥伪造 License。针对这种情况：
- 对产品二进制做代码混淆
- 将公钥分散到多处代码中验证（不集中在一个文件）
- 极端场景可以引入在线激活（联网时验证一次）

---

**Q4：License 到期后产品是直接停用还是降级？**

建议分阶段处理，不要直接停用（会严重影响客户生产）：

| 时间节点 | 行为 |
|---|---|
| 到期前 30 天 | 管理员后台显示到期提醒 |
| 到期前 7 天 | 每次登录提示 |
| 到期当天 | 功能降级（只读模式，不能新增内容）|
| 到期后 30 天 | 完全禁止访问（宽限期） |

宽限期给客户时间续费，避免因商务流程延误影响业务。

---

**Q5：不同客户的功能配置怎么管理？**

每个 .lic 文件的 `features` 字段独立配置。运营后台维护一个客户 License 表，记录每个客户的版本和功能开关。

产品中通过统一的 SDK 判断：

```python
from license import LicenseValidator

validator = LicenseValidator()
if validator.has_feature("ai-inference"):
    # 显示 AI 功能
else:
    # 显示升级提示
```

---

**Q6：如何防止一个 License 文件被多个客户共享使用？**

使用**机器指纹绑定**（可选）：在生成 License 时，将客户服务器的硬件信息（MAC 地址、CPU 序列号等组合哈希）写入 `machine_fingerprint` 字段。产品验证时对比当前机器的指纹。

注意：机器指纹绑定增加了运维成本（换服务器需要重新申请 License），建议仅对高价值客户（企业版）开启，基础版和专业版不绑定。
