# License 授权 — 私钥管理规范

> 私钥是整个 License 体系的信任根，私钥泄漏等同于 License 机制完全失效。

---

## 一、私钥生成

```bash
# 生成 ECDSA P-256 密钥对（推荐，比 RSA 密钥更短）
openssl ecparam -name prime256v1 -genkey -noout -out license_private.pem

# 从私钥提取公钥（嵌入产品使用）
openssl ec -in license_private.pem -pubout -out license_public.pem

# 验证
openssl ec -in license_private.pem -text -noout
```

生成后立即：
1. 将私钥导入 Vault（见下节）
2. **删除本地 `license_private.pem` 文件**（本地不留存）
3. 将公钥 `license_public.pem` 提交到产品代码仓库

---

## 二、私钥存储（HashiCorp Vault）

```bash
# 将私钥存入 Vault KV
vault kv put secret/license/private-key \
  value="$(cat license_private.pem)" \
  created_at="2026-05-28" \
  owner="ops-team"

# License 生成服务读取私钥（通过 Vault Agent 或 SDK）
vault kv get -field=value secret/license/private-key
```

Vault 访问控制策略：

```hcl
# 仅允许 license-generator 服务读取私钥
path "secret/data/license/private-key" {
  capabilities = ["read"]
}
# 禁止任何人直接导出
path "secret/data/license/private-key" {
  denied_parameters = {
    "export" = []
  }
}
```

## 三、私钥备份

| 备份方式 | 要求 |
|---|---|
| Vault Snapshot | 每日自动备份 Vault 数据快照 |
| 离线冷备 | 加密后存储到 U 盘，放入公司保险柜，至少 2 份 |
| 访问记录 | 每次访问私钥都有 Vault Audit Log 记录，定期审查 |

---

## 四、私钥轮转流程

每年执行一次（或发生安全事件时立即执行）：

```bash
# 1. 生成新密钥对
openssl ecparam -name prime256v1 -genkey -noout -out license_private_new.pem
openssl ec -in license_private_new.pem -pubout -out license_public_new.pem

# 2. 用新私钥为所有有效 License 重新签名
python3 scripts/resign_all_licenses.py --new-key license_private_new.pem

# 3. 将新公钥更新到产品代码（需要发布新版本）
# 4. 通知所有客户升级产品版本 + 下发新 .lic 文件
# 5. 将新私钥存入 Vault，旧私钥归档（不删除，用于历史验证）
```

## 五、泄漏应急响应

| 步骤 | 操作 | 时限 |
|---|---|---|
| 1 | 确认泄漏范围（代码仓库/邮件/其他）| 立即 |
| 2 | 在 Vault 中吊销泄漏私钥的访问权限 | 30 分钟内 |
| 3 | 生成新密钥对 | 2 小时内 |
| 4 | 用新私钥为所有有效 License 重新签名 | 4 小时内 |
| 5 | 通知所有客户：需升级产品版本并替换 .lic | 24 小时内 |
| 6 | 事后复盘：泄漏原因 + 预防措施 | 1 周内 |
