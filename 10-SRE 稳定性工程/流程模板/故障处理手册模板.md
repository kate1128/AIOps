## 排查步骤

### 1. 确认告警真实性

```bash
# 查看 Grafana 告警详情，确认不是指标抖动
# 查看近 5 分钟错误率趋势
```

### 2. 查看 Pod 状态

```bash
kubectl get pods -n ai-service
kubectl describe pod <pod-name> | grep -A 20 Events
```

### 3. 查看应用日志

```bash
kubectl logs <pod-name> --tail=100 | grep -E "ERROR|WARN|OOM"
# 或通过 Grafana Loki 查询：
# {namespace="ai-service"} |= "ERROR"
```

### 4. 查看 GPU 状态

```bash
kubectl exec -it <pod-name> -- nvidia-smi
# 或查看 DCGM 指标
curl http://<vllm-svc>:8000/metrics | grep -E "kv_cache|error"
```

### 5. 判断根因

| 现象 | 可能原因 | 下一步 |
|---|---|---|
| KV Cache > 90% | 并发过高 | → 处置方案 A |
| 模型加载时 OOM | max-model-len 过大 | → 处置方案 B |
| Pod CrashLoopBackOff | 启动配置错误 | → 查看 describe 事件 |

---

## 处置方案

### 方案 A（推荐）：重启 Pod

```bash
kubectl rollout restart deployment/<deployment-name> -n ai-service
# 观察重启进度
kubectl rollout status deployment/<deployment-name> -n ai-service
```

预计恢复时间：30-120s（取决于模型大小）

### 方案 B：回滚上一版本

```bash
kubectl rollout undo deployment/<deployment-name> -n ai-service
```

### 方案 C：临时降低并发

```bash
# 修改 vLLM 启动参数，重新部署
# --max-num-seqs 16  （默认 256，临时降低）
```
