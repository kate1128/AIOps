# 告警体系

> 统一告警路由、分组收敛、通知分层——让每个告警被正确的人看到。P1 告警 5 分钟内到达 OnCall，P3 告警每日汇总，不打扰深夜。

## 告警级别定义

| 级别 | 含义 | 响应时限 | 通知方式 | 时间段 |
|------|------|---------|---------|--------|
| **P0** | 生产核心服务不可用 / 数据丢失风险 | 5 分钟 | 飞书 @OnCall + 电话 | 7×24 |
| **P1** | 核心功能降级 / 性能严重劣化 | 30 分钟 | 飞书 @OnCall | 7×24 |
| **P2** | 非核心功能异常 / 预警类 | 工作时间内 | 飞书（不 @all）| 工作时间 |
| **P3** | 优化类告警 / 趋势预警 | 下次迭代 | 每日汇总推送 | 工作时间 |

## Alertmanager 路由配置

```yaml
# alertmanager.yml
global:
  resolve_timeout: 5m

route:
  group_by: ['alertname', 'namespace', 'severity']
  group_wait: 30s       # 等待 30s 聚合同类告警
  group_interval: 5m    # 同组告警最少 5 分钟发一次
  repeat_interval: 4h   # 告警持续时不重复通知（最多 4h 发一次）
  receiver: 'default-feishu'

  routes:
    # P0：立即通知 OnCall
    - match:
        severity: critical
      receiver: 'p0-oncall'
      group_wait: 10s
      repeat_interval: 30m    # P0 每 30 分钟重复提醒（未解决时）

    # P1：工作时间 + 夜间 OnCall
    - match:
        severity: warning
      receiver: 'p1-feishu'
      repeat_interval: 2h

    # P3：每日汇总，不实时打扰
    - match:
        severity: info
      receiver: 'daily-digest'
      group_wait: 1h
      group_interval: 24h
      repeat_interval: 24h

receivers:
  - name: 'default-feishu'
    webhook_configs:
    - url: 'https://open.feishu.cn/open-apis/bot/v2/hook/WEBHOOK_TOKEN'
      send_resolved: true

  - name: 'p0-oncall'
    webhook_configs:
    - url: 'https://open.feishu.cn/open-apis/bot/v2/hook/P0_WEBHOOK_TOKEN'
      send_resolved: true
    # 生产建议：接入 PagerDuty/飞书 OnCall 进行电话呼叫
    pagerduty_configs:
    - service_key: 'PAGERDUTY_SERVICE_KEY'

  - name: 'p1-feishu'
    webhook_configs:
    - url: 'https://open.feishu.cn/open-apis/bot/v2/hook/P1_WEBHOOK_TOKEN'
      send_resolved: true

  - name: 'daily-digest'
    webhook_configs:
    - url: 'http://alert-digest-service:8080/digest'  # 内部汇总服务

inhibit_rules:
  # P0 告警触发时，抑制同命名空间的 P1/P2 告警（防止雪崩）
  - source_match:
      severity: critical
    target_match_re:
      severity: 'warning|info'
    equal: ['namespace', 'alertname']
```

## 关键告警规则库

### 基础设施告警

```yaml
groups:
- name: infrastructure.rules
  rules:
  # 节点 CPU 使用率
  - alert: NodeHighCPU
    expr: (1 - avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m]))) * 100 > 85
    for: 5m
    labels: { severity: warning }
    annotations:
      summary: "节点 CPU 使用率过高: {{ $labels.instance }}"
      description: "当前 CPU 使用率 {{ $value | humanizePercentage }}"

  # 节点内存
  - alert: NodeHighMemory
    expr: (1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 > 90
    for: 5m
    labels: { severity: warning }

  # 节点磁盘
  - alert: NodeDiskAlmostFull
    expr: (1 - node_filesystem_avail_bytes{fstype!="tmpfs"} / node_filesystem_size_bytes) * 100 > 85
    for: 10m
    labels: { severity: warning }

  # Pod 重启
  - alert: PodCrashLooping
    expr: increase(kube_pod_container_status_restarts_total[15m]) > 3
    for: 0m
    labels: { severity: critical }
    annotations:
      summary: "Pod 异常重启: {{ $labels.namespace }}/{{ $labels.pod }}"
```

### AI 推理服务告警

```yaml
- name: ai-inference.rules
  rules:
  # vLLM 推理失败率
  - alert: VLLMHighErrorRate
    expr: rate(vllm:request_success_total{finished_reason="error"}[5m]) /
          rate(vllm:request_success_total[5m]) > 0.05
    for: 2m
    labels: { severity: critical }
    annotations:
      summary: "vLLM 推理失败率超 5%: {{ $labels.model_name }}"

  # 推理 P99 延迟
  - alert: VLLMHighLatency
    expr: histogram_quantile(0.99, rate(vllm:e2e_request_latency_seconds_bucket[5m])) > 10
    for: 5m
    labels: { severity: warning }
    annotations:
      summary: "vLLM P99 延迟超 10s: {{ $labels.model_name }}"

  # GPU 显存使用率
  - alert: GPUMemoryHigh
    expr: DCGM_FI_DEV_FB_USED / (DCGM_FI_DEV_FB_USED + DCGM_FI_DEV_FB_FREE) * 100 > 95
    for: 5m
    labels: { severity: critical }
    annotations:
      summary: "GPU 显存使用率超 95%: GPU {{ $labels.gpu }}"
```
