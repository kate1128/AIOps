# 用户研究规范

> 定义用户研究的方法选择、执行标准和输出规范，确保用户洞察能高质量输入产品决策。

---

## 一、用户分层

### 用户角色定义

| 角色 | 描述 | 典型诉求 | 研究优先级 |
|------|------|---------|-----------|
| **企业管理员** | 负责采购决策和系统配置 | 安全合规、成本效益、易用性 | 高 |
| **一线业务用户** | 日常使用产品完成工作 | 响应速度、准确性、操作流畅 | 高 |
| **IT 运维** | 负责私有化部署和维护 | 部署难度、稳定性、运维工具 | 中 |
| **内容运营** | 维护知识库内容 | 内容管理、更新效率、权限控制 | 中 |

---

## 二、研究方法选择

| 方法 | 适用场景 | 周期 | 产出 |
|------|---------|------|------|
| **用户访谈（深度）** | 理解使用动机、痛点根因 | 需求启动前 / 季度 | 访谈纪要 + 洞察报告 |
| **问卷调研** | 验证假设、量化偏好 | 重大功能发布后 | 数据报告 |
| **可用性测试** | 发现操作流程中的阻力 | 功能原型阶段 | 问题清单 + 录屏 |
| **数据埋点分析** | 发现实际使用路径偏差 | 持续 | 行为漏斗 / 热力图 |
| **客服/投诉分析** | 快速发现高频痛点 | 每周 | TOP 问题列表 |
| **NPS 调研** | 衡量用户满意度和忠诚度 | 季度 | NPS 分值 + 评论分类 |

### 方法选择决策树

```
新功能立项 → 用户访谈（了解真实需求）
功能已有原型 → 可用性测试（发现操作问题）
功能已上线 → 数据分析 + NPS
出现客诉高峰 → 客服分析 + 针对性访谈
```

## 三、用户访谈规范

### 3.1 访谈准备

```
访谈前 3 天：
[ ] 确定访谈目标（本次要回答哪 3 个核心问题）
[ ] 筛选受访用户（符合角色，近 30 天活跃）
[ ] 发送访谈邀请（标注时长 30-45 分钟，给予礼品卡激励）
[ ] 准备访谈大纲（5-8 个开放性问题）

访谈前 1 天：
[ ] 确认用户出席
[ ] 准备录音/录屏设备（需提前获得用户同意）
[ ] 测试线上会议链接
```

### 3.2 访谈大纲模板

```markdown
## 访谈大纲：{研究主题}
研究目标：{需要回答的核心问题}
访谈时长：45 分钟

### 开场（5 min）
- 自我介绍，说明访谈目的
- 获得录音同意
- "没有对错，我们想了解你真实的使用体验"

### 背景了解（10 min）
- 介绍一下你的工作职责？
- 你们公司大概有多少人在用这个产品？
- 你主要用它来做什么？

### 核心问题（20 min）
{根据研究目标定制问题，例如：}
- 描述一下你上次使用{功能}的完整过程？
- 哪个环节让你感觉最困难或最耗时？
- 如果能改变一件事，你最希望改什么？

### 意外发现（5 min）
- 有什么我没有问到，但你觉得很重要的？

### 结束（5 min）
- 感谢，说明后续如何使用访谈结果
```

## 五、NPS 调研规范

### 调研时机

| 触发时机 | 延迟 | 备注 |
|---------|------|------|
| 新用户注册后 30 天 | 30 天后发送 | 给足时间体验产品 |
| 重大版本发布后 | 2 周后 | 收集对新版本的感受 |
| 季度例行调研 | 每季度末 | 全量活跃用户 |

### NPS 计算与分析

```
NPS = 推荐者(9-10分)比例 - 贬损者(0-6分)比例

分级：
- NPS > 50：优秀
- NPS 0-50：良好，需关注改进点
- NPS < 0：警戒，立即深入分析

跟进规则：
- 贬损者（0-6 分）：3 个工作日内联系，了解具体问题
- 被动者（7-8 分）：每季度回访一次
- 推荐者（9-10 分）：邀请成为案例客户或内测用户
```

## 六、研究成果管理

| 产出物 | 存储位置 | 格式 | 有效期 |
|--------|---------|------|--------|
| 访谈纪要 | 本目录 `interviews/` 子目录 | Markdown | 长期 |
| 调研数据报告 | 本目录 `surveys/` 子目录 | Markdown + 图表 | 长期 |
| 用户画像文档 | 本目录 `personas/` 子目录 | Markdown | 每半年 review |
| NPS 追踪 | 飞书多维表格 | 表格 | 持续更新 |

所有研究发现**必须在 1 周内完成整理**，否则细节流失，价值大幅下降。
