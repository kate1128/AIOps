# 市场洞察规范

> 追踪行业趋势、技术动向和政策环境，为产品战略和路线图决策提供外部视角输入。

---

| 维度 | 内容 | 更新频率 |
|------|------|---------|
| **AI 技术趋势** | 大模型进展、RAG/Agent 技术、多模态能力演进 | 月度 |
| **企业知识管理市场** | 市场规模、增速、主要赛道、融资动向 | 季度 |

| **技术生态** | 开源模型、向量数据库、推理框架的演进 | 月度 |

## 三、技术趋势雷达

每半年更新一次技术趋势雷达，评估技术成熟度和采用建议。

### 象限定义

```
                 高商业价值
                      │
    观望（Watch）      │    试点（Trial）
                      │
高不确定性 ──────────────────── 低不确定性
                      │
    忽略（Ignore）     │    采用（Adopt）
                      │
                 低商业价值
```

### 2026 H1 技术雷达示例

| 技术/工具 | 象限 | 建议 | 说明 |
|---------|------|------|------|
| RAG（检索增强生成）| 采用 | 已成熟，继续深化 | 核心竞争力 |
| AI Agent（多步推理）| 试点 | 选定场景验证 | 能力快速演进中 |
| 多模态（图文问答）| 试点 | 评估客户需求 | 模型成熟度提升 |
| 思维链（CoT）| 采用 | 标准做法 | |
| 向量数据库 Milvus | 采用 | 已在用，继续优化 | |
| GraphRAG（知识图谱+RAG）| 观望 | 追踪进展 | 复杂度高，效果不稳定 |
| 模型蒸馏/量化 | 试点 | 评估推理成本效益 | |
| 联邦学习 | 忽略 | 当前无实际需求 | |

## 四、行业报告分析流程

### 获取后的标准处理步骤

```
1. 阅读执行摘要（Executive Summary）
2. 提取与我们相关的核心数据点（3-5 个）
3. 记录到《市场数据台账》（飞书多维表格）
4. 如有影响产品路线图的结论，同步到季度规划
```

### 市场数据台账格式

| 数据点 | 数值 | 来源 | 时间 | 备注 |
|--------|------|------|------|------|
| 中国企业知识管理软件市场规模 | xx 亿元 | IDC 2025 | 2025-Q4 | CAGR xx% |
| AI 问答准确率基准 | xx% | 内部测试 | 2026-Q1 | |

## 五、政策监控

### 重点关注的政策领域

| 领域 | 监控来源 | 影响方向 |
|------|---------|---------|
| 生成式 AI 管理规定 | 国家互联网信息办公室 | 内容安全要求、备案要求 |
| 数据安全法/个人信息保护法 | 法律库 | 数据存储和处理规范 |
| 等保 2.0 | 公安部网安局 | 私有化交付合规要求 |
| 行业 AI 应用规范 | 各行业主管部门 | 金融/医疗等行业特殊要求 |
