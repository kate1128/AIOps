# 技术选型规范

> 标准化的技术选型流程，确保选型有据可依、有记录可查。

---

## 一、选型评估矩阵

| 维度 | 权重 | 评分说明(1-5) |
|------|------|--------------|
| **社区活跃度** | 20% | GitHub Stars、Commit 频率、Release 周期 |
| **技术成熟度** | 25% | 版本号(v1.0+)、生产案例、企业支持 |
| **团队熟悉度** | 15% | 团队已有经验、学习曲线、培训成本 |
| **生态集成** | 20% | 与现有技术栈兼容性、插件/扩展 |
| **License** | 10% | 商业友好、无传染性 License |
| **性能/安全** | 10% | 基准测试、已知 CVE、安全记录 |

---

## 二、技术雷达

```
            ╭──────────╮
            │  核心     │ ← 团队熟练掌握，推荐使用
            │  Adopt    │
            ├──────────┤
            │  试用     │ ← 已验证可行，正在推广
            │  Trial    │
            ├──────────┤
            │  评估     │ ← 值得关注，需要验证
            │  Assess   │
            ├──────────┤
            │  暂停     │ ← 不建议使用，或逐步淘汰
            │  Hold     │
            ╰──────────╯
```

---

## 三、选型记录

每次技术选型完成后，输出选型报告：
- 选型背景和需求
- 候选技术列表（至少 3 个）
- 评估矩阵得分
- 最终决策及理由
- 选型日期和决策人
