# 环境分级规范

> 明确各环境定义、准入标准、访问控制与差异策略，解决当前环境边界模糊、权限混乱的问题。

---

## 一、环境定义与定位

| 环境 | 代号 | 用途 | 对应集群 | 稳定性要求 |
|------|------|------|----------|-----------|
| 开发环境 | Dev | 日常开发自测、联调，允许频繁变更和故障 | Dev（自建） | 较低，允许短时不可用 |
| 测试环境 | Test | QA 功能测试、自动化测试，非正式发布验证 | Dev（自建）或独立命名空间 | 中等，保证工作日可用 |
| 预发布环境 | Pre/Staging | 发布前最后验证，与生产配置尽可能一致 | Pre（自建） | 较高，模拟生产 |
| 生产环境 | Prod | 线上客户使用，最高稳定性要求 | Prod（阿里云 ACK） | 极高，SLA 99.9%+ |

### 可选补充环境

| 环境 | 适用场景 |
|------|---------|
| LoadTest/Perf | 性能压测环境，需与生产等规格 |
| DR/Standby | 容灾备用环境，定期同步生产数据 |

## 二、准入标准

### 2.1 各环境部署准入

```
Dev:
  - 代码可编译通过
  - 单元测试通过（可选）
  - 无需审批，开发者自行触发

Test:
  - 代码 MR 已创建
  - 单元测试通过（强制）
  - CI 流水线通过
  - QA 确认部署（可选）

Pre:
  - Test 环境验证通过
  - 集成测试 / E2E 测试通过
  - API 兼容性检查通过
  - 变更审批通过（发布审批人）
  - 变更窗口：工作日 10:00-16:00

Prod:
  - Pre 环境验证通过（至少运行 24h 无严重问题）
  - 性能/稳定性测试通过（涉及性能变更时）
  - 回滚方案就绪
  - 发布审批通过（至少 2 人：技术负责人 + 产品/PM）
  - 变更窗口：周二/四 14:00-16:00（禁止周五及节假日前发布）
```

### 2.2 环境数据隔离

| 数据类型 | Dev | Test | Pre | Prod |
|---------|-----|------|-----|------|
| 业务数据库 | 脱敏/空数据 | 脱敏/有限真实数据 | 脱敏副本 | 真实数据 |
| 缓存 | 独立 Redis | 独立 Redis | 独立 Redis | 独立 Redis |
| 对象存储 | 独立 Bucket | 独立 Bucket | 独立 Bucket | 独立 Bucket |
| AI 模型 | 小模型/精简版 | 精简版 | 生产镜像 | 生产模型 |
| 密钥/凭据 | 开发 Key | 开发 Key | 预发 Key | 生产 Key |
| 日志 | 不保留 | 保留 7 天 | 保留 15 天 | 保留 30+ 天 |

> **严禁 Pre 环境直连 Prod 数据库或混用存储。**

## 三、访问控制

### 3.1 人员权限

| 角色 | Dev | Test | Pre | Prod |
|------|-----|------|-----|------|
| 开发者 | 读写 | 读 | 无 | 无 |
| QA | 读 | 读写 | 读 | 无 |
| SRE/运维 | 读写 | 读写 | 读写 | 读写（需审批）|
| 技术负责人 | 读写 | 读写 | 读写 | 只读（紧急时读写）|
| 产品/PM | 无 | 无 | 只读 | 只读（只读账号）|

### 3.2 网络访问

| 访问方式 | Dev | Test | Pre | Prod |
|---------|-----|------|-----|------|
| 集群 API Server | VPN + IP 白名单 | VPN + IP 白名单 | VPN + IP 白名单 | 堡垒机 + IP 白名单 |
| 服务 Ingress | VPN 内网 | VPN 内网 | VPN 内网 | CDN/WAF + 公网 |
| 数据库直连 | 禁止 | 禁止 | 禁止 | 禁止（仅应用层访问）|
| SSH 节点 | 运维组 | 运维组 | 运维组 | 堡垒机审计 |

### 3.3 操作审计

```yaml
Prod 环境操作原则：
  - 所有 CLI 操作必须通过堡垒机（JumpServer）
  - kubectl 使用只读 ServiceAccount，变更走 CI/CD
  - 紧急操作需录屏 + 事后复盘
  - 操作日志保留 ≥ 180 天

Pre 环境操作原则：
  - 推荐通过堡垒机操作
  - 允许运维人员直接 SSH（通知即可）
  - 操作日志保留 ≥ 90 天

Dev/Test 环境操作原则：
  - 无强制审计要求
  - 禁止将 Prod 凭据用于 Dev 环境
```

## 四、环境差异策略

### 4.1 资源规格差异

| 资源类型 | Dev | Test | Pre | Prod |
|---------|-----|------|-----|------|
| 副本数 | 1 | 1-2 | 2 | 3+（跨 AZ）|
| CPU Request | 0.5x 生产 | 0.5x 生产 | 1x 生产 | 1x 生产 |
| Memory Request | 0.5x 生产 | 0.5x 生产 | 1x 生产 | 1x 生产 |
| GPU 卡数 | 1/n 卡 | 1 卡 | 1-2 卡 | 按需 |
| HPA | 关闭 | 关闭 | 开启（低阈值）| 开启 |

### 4.2 监控与告警差异

| 监控项 | Dev | Test | Pre | Prod |
|--------|-----|------|-----|------|
| 基础指标（CPU/Mem） | 采集 | 采集 | 采集 | 采集 + 告警 |
| 业务指标 | 不采集 | 选择性采集 | 采集 | 采集 + 告警 |
| 日志采集 | 控制台 | 采集（7天）| 采集（15天）| 采集（30天）|
| 链路追踪 | 关闭 | 采样 10% | 采样 50% | 采样 100%（错误）+ 10%（正常）|
| 告警通知 | 无 | 仅 Warning | Warning + 部分 Critical | Critical 即时通知 |

### 4.3 备份策略差异

| 备份项 | Dev | Test | Pre | Prod |
|--------|-----|------|-----|------|
| etcd | 无 | 无 | 每日备份，保留 7 天 | 每日备份，保留 30 天 |
| 数据库 | 无 | 无 | 每日备份 | 每日全量 + WAL 归档 |
| 持久卷 | 无 | 无 | 按需 | 自动快照 |
| K8s 资源清单 | 无 | 无 | 每周导出 | 每日导出 |

## 五、环境命名规范

### 5.1 K8s Namespace 命名

```
格式：<应用/团队>-<环境>
示例：
  smartvision-prod
  smartvision-pre
  smartvision-test
  ai-infra-prod
  middleware-prod
```

### 5.2 K8s Context 命名

```
格式：<集群名称>-<环境>
示例：
  ack-prod
  self-pre
  self-dev
```

### 5.3 标签规范

```yaml
# 所有资源必须携带 env 标签
labels:
  env: prod    # prod | pre | test | dev
  team: ai-infra
  app: vllm-qwen-7b
```

## 六、环境生命周期管理

| 阶段 | 动作 | 说明 |
|------|------|------|
| 创建 | 申请审批 | 提交环境需求，SRE 评估资源后创建 |
| 日常 | 健康巡检 | 每周自动巡检，检查节点/Pod/证书状态 |
| 降级 | 回收资源 | 长期闲置（30天无变更）环境自动缩容通知 |
| 销毁 | 数据备份后回收 | 项目终止时，数据备份归档后销毁环境 |
