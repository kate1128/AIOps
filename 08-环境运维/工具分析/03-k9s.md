# k9s — Kubernetes 终端 UI

> 键盘驱动的 K8s 终端管理工具，SSH 环境首选，熟练后效率超过 GUI。

---

## 是什么

k9s 是一个基于终端的 Kubernetes 管理 UI（TUI），运行在命令行里，完全键盘操作，零依赖、零服务端安装。它对 kubectl 命令的常见操作（查看 Pod、进容器、查日志、删资源）做了交互式封装，极大提升操作效率。

**核心特点：**
- 纯终端，SSH 进服务器后直接运行，不需要本地 GUI 环境
- 响应极快，大集群下不卡顿
- 支持自定义快捷键和皮肤
- 完全开源，无商业限制

---

## 核心能力

| 能力 | 快捷键 | 说明 |
|---|---|---|
| 资源导航 | `:pod`、`:svc`、`:deploy` | 输入资源类型跳转，支持所有 K8s 资源 |
| 实时日志 | `l` | 进入 Pod 详情后按 l 查看日志，`-f` 跟随 |
| 进入容器 | `s` | 直接 exec 进 Shell，多容器 Pod 支持选择 |
| 删除资源 | `ctrl+d` | 带确认提示，防误删 |
| 描述资源 | `d` | 等同于 `kubectl describe`，格式化展示 |
| 查看 YAML | `y` | 直接看资源 YAML |
| 编辑资源 | `e` | 调用 `$EDITOR` 编辑并 apply |
| 端口转发 | `shift+f` | 交互式 Port Forward，无需手打命令 |
| 过滤搜索 | `/` | 实时过滤资源名，支持正则 |
| 命名空间切换 | `:ns` 或配置别名 | 快速切换 namespace |
| Context 切换 | `:ctx` | 切换到不同集群 |
| Pulse 视图 | `:pulse` | 集群整体健康状态总览 |
| Xray 视图 | `:xray deploy` | 展示 Deployment → Pod → Container 层级树 |
| Popeye 扫描 | `:popeye` | 内置集群最佳实践扫描（资源配置问题）|

## 安装

```bash
# macOS
brew install derailed/k9s/k9s

# Linux（二进制，推荐，直接在服务器上用）
wget https://github.com/derailed/k9s/releases/latest/download/k9s_Linux_amd64.tar.gz
tar -xzf k9s_Linux_amd64.tar.gz
mv k9s /usr/local/bin/

# Windows
choco install k9s
# 或下载 zip：https://github.com/derailed/k9s/releases

# 验证
k9s version
```

### 多集群配置

```bash
# 使用指定 kubeconfig 启动
k9s --kubeconfig ~/.kube/prod

# 使用指定 context
k9s --context prod-cluster

# 启动时指定 namespace
k9s -n smartvision-prod
```

## 与本项目的关系

你们有 3 套集群，日常需要频繁切换。k9s 的使用场景：

```
运维人员 SSH 到跳板机 / 本地终端
    └── k9s --context prod      # 生产集群（快速查问题）
    └── k9s --context pre       # pre 环境（部署验证）
    └── k9s --context dev       # dev 环境（日常开发）
```

**特别推荐场景：**
- 生产环境排障：SSH 进跳板机，直接 k9s 查 Pod 日志和 Event，比本地打开 Lens 快
- 查 NotReady 节点：`:node` → 找到问题节点 → `d` 看详情
- 查 AI 服务 Pod：`:pod` → `/llm` 过滤 AI 相关 Pod → 查资源使用
- 紧急删除僵尸 Pod：定位后 `ctrl+d` 删除，不容易误操作

### 与 Lens 配合使用

| 场景 | 推荐工具 |
|---|---|
| 本地日常浏览、新成员查看 | Lens（GUI 友好）|
| SSH 环境、生产排障、高频操作 | k9s（键盘效率更高）|
| 需要看资源消耗图表 | Lens（Metrics 可视化）|
| 快速过滤大量 Pod | k9s（`/` 正则过滤更快）|

## 常用操作速查

```bash
# 进入 k9s
k9s

# 常用视图快捷键
:pod          # 查看所有 Pod
:svc          # 查看 Service
:deploy       # 查看 Deployment
:node         # 查看节点
:ev           # 查看 Events（排障必看）
:pvc          # 查看持久卷
:cm           # 查看 ConfigMap
:ing          # 查看 Ingress

# 在资源列表中
/             # 搜索过滤
l             # 查看日志
s             # 进入容器 Shell
d             # describe
y             # 查看 YAML
e             # 编辑 YAML
ctrl+d        # 删除（带确认）
shift+f       # 端口转发

# 切换命名空间
:ns           # 列出所有 namespace，选择后切换

# 退出
:quit 或 ctrl+c
```

## 局限性

- 纯终端，不适合不熟悉键盘操作的人员
- 没有图表/趋势视图（需要 Grafana 补充）
- 编辑资源时依赖 `$EDITOR` 变量配置（默认 vi）
- Windows 终端下部分快捷键可能冲突
