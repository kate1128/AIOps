# 云资源成本规范

> 定义云资源打标、成本可视化和月度账单 Review 流程，实现成本可归因、可追踪、可优化。

---

## 一、资源打标（Tag）规范

所有云资源**必须**打以下标签，否则不计入成本归因，统一归入"未标记"类别并发起整改。

### 必填标签

| 标签键 | 示例值 | 说明 |
|--------|--------|------|
| `env` | `prod` / `pre` / `dev` | 环境类型 |
| `service` | `ai-backend` / `scheduler` / `frontend` | 服务名称，与代码仓库一致 |
| `team` | `ai-infra` / `backend` / `platform` | 负责团队 |
| `cost-center` | `r&d` / `ops` / `delivery` | 成本中心（对应预算归属）|

### 可选标签

| 标签键 | 示例值 | 说明 |
|--------|--------|------|
| `project` | `feature-rag-v2` | 特定项目，临时资源使用 |
| `customer` | `customer-a` | 私有化交付资源 |
| `ttl` | `2026-08-01` | 临时资源的过期时间 |

### 打标检查

```bash
# 查找阿里云 ECS 中未打必填标签的实例（示例命令）
aliyun ecs DescribeInstances --RegionId cn-shanghai \
  --output cols=InstanceId,InstanceName,Tags --rows

# K8s 资源打标签（Deployment 示例）
kubectl label deployment ai-backend \
  env=prod service=ai-backend team=ai-infra cost-center=r&d -n prod
```

## 二、成本分摊模型

### 分摊维度

```
总成本
├── 按环境分摊
│   ├── prod（生产）：~70%
│   ├── pre（预发）：~20%
│   └── dev（开发）：~10%
├── 按服务分摊
│   ├── AI 推理服务（GPU 节点为主）：~75%
│   ├── 后端服务（CPU 节点）：~15%
│   └── 基础设施（存储、网络、监控）：~10%
└── 按团队分摊
    ├── AI Infra 团队
    └── 平台/后端团队
```

### Kubecost 接入

```bash
# 安装 Kubecost（免费版）
helm repo add kubecost https://kubecost.github.io/cost-analyzer
helm install kubecost kubecost/cost-analyzer \
  --namespace kubecost --create-namespace \
  --set kubecostToken="YOUR_TOKEN" \
  --set prometheus.enabled=false \
  --set prometheus.fqdn=http://prometheus-server.observability.svc:80

# 访问 Kubecost UI
kubectl port-forward svc/kubecost-cost-analyzer 9090 -n kubecost
# 浏览器打开 http://localhost:9090
```

### 成本归因核心指标

| 指标 | 查看方式 | 更新频率 |
|------|---------|---------|
| 按 Namespace 的月成本 | Kubecost → Cost Allocation | 每日 |
| 按 Deployment 的成本趋势 | Kubecost → Assets | 每日 |
| 未标记资源成本 | Kubecost → Savings → Unallocated | 每日 |
| GPU 节点每小时费用 | 阿里云账单明细 | 月度 |

## 四、预算管理

### 预算设置

```bash
# 阿里云预算告警设置（控制台操作步骤）
# 1. 费用中心 → 预算管理 → 新建预算
# 2. 预算类型：月度预算
# 3. 设置金额和告警阈值（建议：50%/80%/100% 三级告警）
# 4. 告警通知：邮件 + 飞书 Webhook
```

### 预算告警响应

| 告警级别 | 触发条件 | 响应动作 |
|---------|---------|---------|
| 提醒 | 消耗达预算 50% | 运维确认趋势是否正常 |
| 警告 | 消耗达预算 80% | 运维分析超支原因，评估是否需要调整 |
| 紧急 | 消耗达预算 100% | 立即暂停非必要资源，向管理层报告 |

## 五、私有化交付成本隔离

私有化交付给客户的资源需要单独核算：

```bash
# 为客户专用资源打标
aliyun ecs AddTags --ResourceType instance \
  --ResourceId i-xxxxxxxx \
  --Tags '[{"Key":"customer","Value":"customer-a"},{"Key":"cost-center","Value":"delivery"}]'
```

客户资源成本用于计算**每客户交付成本**，纳入私有化交付项目的盈亏分析。
