# 离线包管理规范

> 定义私有化交付离线安装包的构建、测试、分发和版本管理流程，解决客户内网隔离场景下的交付问题。

---

## 一、离线包适用场景

| 场景 | 描述 |
|------|------|
| **完全离网** | 客户服务器无任何公网访问能力 |
| **受限网络** | 可访问部分网络但无法访问 Docker Hub / Harbor |
| **安全合规要求** | 客户安全策略要求所有软件包来源可控、可审计 |

---

## 二、离线包内容组成

每个版本的离线安装包应包含：

```
wenxue-{version}-offline/
├── images/                    # 所有 Docker 镜像（tar 格式）
│   ├── ai-backend-v1.2.3.tar
│   ├── scheduler-v1.2.3.tar
│   ├── frontend-v1.2.3.tar
│   ├── postgres-15.tar
│   ├── redis-7.tar
│   ├── milvus-2.4.tar
│   └── ...（所有依赖镜像）
├── models/                    # AI 模型文件（可选，体积大时单独提供）
│   └── README.md              # 说明如何单独获取模型
├── charts/                    # Helm Charts（如有）
├── playbooks/                 # Ansible 脚本
│   └── install.sh             # 一键安装入口
├── configs/                   # 配置模板
│   └── inventory/
│       └── template.yml
├── tools/                     # 辅助工具（kubectl、helm 二进制）
│   ├── kubectl-linux-amd64
│   └── helm-linux-amd64
├── checksums.sha256           # 所有文件的 SHA256 校验值
└── RELEASE_NOTES.md           # 版本说明
```

## 三、离线包构建流程

### 3.1 自动化构建脚本

```bash
#!/bin/bash
# build-offline-package.sh
# 使用方式：bash build-offline-package.sh v2.1.0

set -e

VERSION=${1:?请指定版本号，如 v2.1.0}
OUTPUT_DIR="wenxue-${VERSION}-offline"
REGISTRY="harbor.internal.com"

echo "📦 构建离线包：$VERSION"
mkdir -p $OUTPUT_DIR/images

# 1. 拉取所有镜像
IMAGES=(
  "${REGISTRY}/prod/ai-backend:${VERSION}"
  "${REGISTRY}/prod/scheduler:${VERSION}"
  "${REGISTRY}/prod/frontend:${VERSION}"
  "postgres:15-alpine"
  "redis:7-alpine"
  "milvus/milvus:v2.4.0"
  "minio/minio:latest"
  "nginx:1.25-alpine"
)

for image in "${IMAGES[@]}"; do
  name=$(echo $image | tr '/:' '-')
  echo "📥 拉取：$image"
  docker pull $image
  echo "💾 保存：$name.tar"
  docker save $image > "$OUTPUT_DIR/images/$name.tar"
done

# 2. 复制脚本和配置
cp -r playbooks/ $OUTPUT_DIR/
cp -r configs/ $OUTPUT_DIR/

# 3. 复制工具
mkdir -p $OUTPUT_DIR/tools
cp /usr/local/bin/kubectl $OUTPUT_DIR/tools/kubectl-linux-amd64
cp /usr/local/bin/helm $OUTPUT_DIR/tools/helm-linux-amd64

# 4. 生成校验文件
echo "🔑 生成 SHA256 校验..."
find $OUTPUT_DIR -type f | sort | xargs sha256sum > $OUTPUT_DIR/checksums.sha256

# 5. 打包压缩
echo "🗜️ 压缩打包..."
tar -czf "${OUTPUT_DIR}.tar.gz" $OUTPUT_DIR

echo "✅ 完成：${OUTPUT_DIR}.tar.gz"
echo "📊 大小：$(du -sh ${OUTPUT_DIR}.tar.gz | cut -f1)"
```

### 3.2 构建环境要求

- 构建机需能访问 Harbor 和公网（用于拉取基础镜像）
- 磁盘空间：至少 50GB（镜像未压缩时较大）
- 构建后的 tar.gz 大小参考：标准版 ~5GB，AI 版 ~15GB（不含模型）

### 3.3 模型文件单独处理

AI 模型文件（如 qwen-7b、embedding 模型）体积过大，不打入主包：

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| 随包附带（U 盘）| 客户完全离线 | 需要额外的大容量 U 盘（模型 ~20GB/个）|
| 客户自行下载 | 客户有受限外网 | 提供模型下载链接和校验值 |
| 实施工程师携带 | 小规模交付 | 笔记本/硬盘携带 |

## 四、离线包测试

每次发布前**必须**在隔离环境中完成测试：

### 测试清单

```
环境：一台/一组无公网访问的 Linux 服务器（模拟客户环境）

[ ] 解压安装包无报错
[ ] 镜像加载成功（docker load < image.tar）
[ ] 执行 install.sh，全程无需访问公网
[ ] preflight 检查通过
[ ] 所有服务启动成功，Health Check 通过
[ ] 功能验收：能正常登录、上传文档、发起查询
[ ] 离线版本号与包名一致
[ ] checksums.sha256 验证通过
```

### 测试命令

```bash
# 校验包完整性
sha256sum -c checksums.sha256

# 批量加载镜像
for tar_file in images/*.tar; do
  echo "加载：$tar_file"
  docker load < "$tar_file"
done

# 安装
bash install.sh
```

## 五、离线包分发

### 分发渠道

| 渠道 | 适用场景 | 安全要求 |
|------|---------|---------|
| 阿里云 OSS 私有桶（时效链接）| 客户有互联网访问 | 时效签名 URL，24-72h 有效 |
| 实施工程师随身携带（加密 U 盘）| 完全离线客户 | U 盘加密（VeraCrypt）|
| 客户指定 FTP/SCP | 客户有内网传输通道 | 传输后用 checksums.sha256 校验 |

### 下载方式示例（OSS 时效链接）

```bash
# 生成时效链接（运维执行）
aliyun oss sign --endpoint oss-cn-shanghai.aliyuncs.com \
  oss://wenxue-delivery/wenxue-v2.1.0-offline.tar.gz \
  --timeout 86400

# 客户下载后验证
sha256sum wenxue-v2.1.0-offline.tar.gz
# 对比官方提供的 SHA256 值
```

## 六、离线包版本管理

| 版本 | 发布日期 | 大小 | 状态 | 支持截止 |
|------|---------|------|------|---------|
| v2.1.0 | 待填写 | - | 当前版本 | - |

保留规则：
- 当前版本（Current）：永久保留
- 最近 3 个版本：保留 1 年
- 更早版本：归档到冷存储，不主动分发但保留用于应急
