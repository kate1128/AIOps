# 环境检查规范（preflight）

> 部署前必须执行的环境检查清单，全部通过才允许进入部署阶段。

---

## 检查项总表

| 编号 | 检查项 | 标准 | 失败行为 |
|---|---|---|---|
| F01 | CPU 核数 | ≥ 16 核 | ❌ 阻止部署 |
| F02 | 内存 | ≥ 64 GB | ❌ 阻止部署 |
| F03 | /data 磁盘剩余 | ≥ 200 GB | ❌ 阻止部署 |
| F04 | 操作系统 | Ubuntu 22.04 / CentOS 7+ | ⚠️ 警告，需确认 |
| F05 | Docker 版本 | ≥ 20.10 | ❌ 阻止部署 |
| F06 | Python 版本 | ≥ 3.8 | ❌ 阻止部署 |
| F07 | 端口 80 | 未被占用 | ❌ 阻止部署 |
| F08 | 端口 443 | 未被占用 | ❌ 阻止部署 |
| F09 | 端口 3306（MySQL）| 未被占用 | ❌ 阻止部署 |
| F10 | 端口 6379（Redis）| 未被占用 | ❌ 阻止部署 |
| F11 | 端口 9200（ES）| 未被占用 | ❌ 阻止部署 |
| F12 | 节点间网络互通 | ping 延迟 < 5ms | ❌ 阻止部署 |
| F13 | NTP 时间同步 | 误差 < 1 秒 | ⚠️ 警告 |
| F14 | GPU 驱动（AI 版）| nvidia-smi 可用 | ❌ 阻止部署 |
| F15 | CUDA 版本（AI 版）| ≥ 12.0 | ❌ 阻止部署 |
| F16 | `vm.max_map_count` | ≥ 262144（ES 要求）| ❌ 阻止部署 |

---

## 手动快速检查命令

在客户服务器上逐条执行，部署前自检：

```bash
# F01-F03：硬件
nproc
free -h
df -h /data

# F04：OS
cat /etc/os-release | grep PRETTY_NAME

# F05：Docker
docker --version

# F06：Python
python3 --version

# F07-F11：端口
ss -tlnp | grep -E ':80|:443|:3306|:6379|:9200'
# 有输出说明端口被占用

# F12：节点互通（在 app-server 上 ping db-server）
ping -c 3 {db-server-ip}

# F13：NTP
timedatectl show | grep NTPSynchronized

# F14-F15：GPU（AI 版）
nvidia-smi
nvcc --version

# F16：内核参数
sysctl vm.max_map_count
# 如果小于 262144，执行：sysctl -w vm.max_map_count=262144
```

## Ansible 自动检查（check.yml）

> 目标态：上面的检查全部由 check.yml 自动完成，失败时输出清晰的错误信息。

```yaml
# roles/preflight/tasks/main.yml 核心逻辑
- name: 检查 CPU 核数
  assert:
    that: ansible_processor_vcpus >= 16
    fail_msg: "CPU 核数不足：当前 {{ ansible_processor_vcpus }} 核，需要 ≥ 16 核"

- name: 检查内存
  assert:
    that: ansible_memtotal_mb >= 65536
    fail_msg: "内存不足：当前 {{ (ansible_memtotal_mb / 1024) | int }} GB，需要 ≥ 64 GB"

- name: 检查 /data 磁盘
  shell: df /data --output=avail | tail -1
  register: disk_avail
  failed_when: disk_avail.stdout | int < 209715200  # 200GB in KB

- name: 检查端口 3306
  wait_for:
    port: 3306
    state: stopped
    timeout: 3
  ignore_errors: yes
  register: port_3306
- name: 确认 3306 未被占用
  fail:
    msg: "端口 3306 已被占用，请确认是否有其他 MySQL 实例运行"
  when: port_3306 is not failed

- name: 检查 GPU 驱动（AI 版）
  command: nvidia-smi
  register: nvidia_check
  failed_when: nvidia_check.rc != 0
  when: enable_ai | default(false)
```
