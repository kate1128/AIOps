# 实施手册

> 实施工程师执行私有化交付的 SOP，新人看完本文档应能独立完成一次标准版交付。

---

## 一、交付前准备（T-1 天）

### 1.1 收集客户环境信息

发给客户填写的确认表（邮件或飞书文档）：

```
【问学 2.0 部署环境确认表】

服务器信息：
- 服务器数量：___台
- 每台规格：___核 CPU / ___GB 内存 / ___GB 磁盘
- 操作系统：Ubuntu 22.04 / CentOS 7（圈选）
- 是否含 GPU：是 / 否；如有：GPU 型号___ 数量___

网络信息：
- 各服务器 IP：___
- 是否可访问公网：是 / 否
- 客户期望域名：___

已有服务（避免端口冲突）：
- 80/443 端口是否被占用：是 / 否
- MySQL 是否自备：是 / 否
- Redis 是否自备：是 / 否
```

### 1.2 填写 inventory 文件

```bash
cd playbooks/
cp inventory/template.yml inventory/{customer-name}.yml
vim inventory/{customer-name}.yml
# 填写客户服务器 IP、域名、密码等
```

### 1.3 测试 SSH 连通性

```bash
ansible all -i inventory/{customer-name}.yml -m ping
# 所有主机应返回 pong
```

## 二、实施阶段（当天）

### 2.1 执行 preflight 检查

```bash
ansible-playbook -i inventory/{customer-name}.yml playbooks/check.yml
```

**如果有报错**：
- `端口占用`：要求客户停用或配合改端口
- `磁盘不足`：要求扩容或清理，最少需要 200 GB
- `Docker 版本过低`：执行 `roles/preflight/files/upgrade-docker.sh` 升级

检查全部通过后，再进行部署。**不允许跳过 preflight 直接部署。**

### 2.2 一键部署

```bash
# 标准版（不含 AI）
ansible-playbook -i inventory/{customer-name}.yml playbooks/site.yml

# AI 版
ansible-playbook -i inventory/{customer-name}.yml playbooks/site.yml \
  -e "enable_ai=true"

# 加 -v 查看详细输出（排障时使用）
ansible-playbook -i inventory/{customer-name}.yml playbooks/site.yml -v
```

部署耗时约 15-30 分钟，期间可观察输出。如有任务失败，记录报错 task 名称，对照 FAQ.md 处理。

### 2.3 配置初始化

部署完成后，登录管理后台初始化：

```bash
# 1. 获取初始管理员密码
cat /data/wenxue/.admin-password

# 2. 登录后台
http://{app_domain}/admin

# 3. 修改管理员密码（必须在交付前完成）
# 4. 配置客户 Logo 和系统名称
# 5. 创建初始用户账号
```

## 三、验收阶段

### 3.1 执行自动验收脚本

```bash
ansible-playbook -i inventory/{customer-name}.yml playbooks/verify.yml
```

脚本自动检查所有核心接口，并在 `/tmp/` 生成验收报告文件：
`delivery-report-{version}-{date}.md`

### 3.2 人工确认项

自动验收通过后，和客户确认以下项（约 30 分钟）：

- [ ] 登录主页正常显示
- [ ] 创建一条内容，保存成功
- [ ] AI 对话功能正常（AI 版）
- [ ] 管理员可以新增用户
- [ ] 客户提供的域名 HTTPS 证书生效

### 3.3 交付物清单

移交给客户：

- [ ] 验收报告（`delivery-report-*.md`）
- [ ] 管理员账号和初始密码
- [ ] `inventory/{customer-name}.yml`（脱敏版，密码替换为 `***`）
- [ ] 运维手册（常用操作命令）

---

## 四、回滚操作

```bash
# 全量回滚到上一版本
ansible-playbook -i inventory/{customer-name}.yml playbooks/rollback.yml

# 指定回滚某个组件
ansible-playbook -i inventory/{customer-name}.yml playbooks/rollback.yml \
  -e "rollback_component=app"
```
