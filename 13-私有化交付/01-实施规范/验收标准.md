# 验收标准

> 什么叫"交付成功"的明确定义。自动验收脚本 + 人工确认双重保障。

---

## 一、核心接口验收（自动）

以下接口必须全部返回 200，否则验收不通过：

| 接口 | 方法 | 期望响应 | 说明 |
|---|---|---|---|
| `/api/health` | GET | `{"status":"ok"}` | 主服务健康检查 |
| `/api/v1/auth/login` | POST | HTTP 200 + token | 用账号密码登录 |
| `/api/v1/content/list` | GET | HTTP 200 + 列表数据 | 内容列表接口 |
| `/api/ai/v1/models` | GET | HTTP 200 + 模型列表 | AI 版专属 |
| `/` | GET | HTTP 200 | 前端主页加载 |

---

## 二、性能基线（自动）

| 指标 | 标准 | 测试方法 |
|---|---|---|
| 登录接口 P99 延迟 | < 500ms | `ab -n 100 -c 10 /api/v1/auth/login` |
| 内容列表 P99 延迟 | < 1000ms | 同上 |
| AI 推理首 Token 延迟（AI 版）| < 3s | curl 计时 |
| 页面首屏加载时间 | < 3s | Chrome DevTools Network |

---

## 三、人工确认项

实施工程师和客户代表共同确认（约 30 分钟）：

### 基础功能

- [ ] 使用管理员账号正常登录
- [ ] 创建一条内容，保存成功，刷新后数据仍在
- [ ] 文件上传功能正常（上传一张图片）
- [ ] 用户管理：新增一个普通用户，用该账号登录成功

### AI 功能（AI 版必须）

- [ ] AI 对话界面正常打开
- [ ] 发送一条消息，AI 正常回复（延迟 < 10s）
- [ ] 切换模型（如有多个模型）正常

### 系统配置

- [ ] 域名 HTTPS 证书正常（浏览器地址栏无安全警告）
- [ ] 管理员初始密码已修改为客户自定义密码
- [ ] 系统名称已配置为客户品牌名称

## 四、交付物清单

交付给客户的文件，缺一不给：

| 文件 | 内容 | 格式 |
|---|---|---|
| 验收报告 | 所有接口测试结果 + 性能数据 | Markdown / PDF |
| 管理员账号信息 | 账号 + 初始密码 | 加密文档 |
| inventory 脱敏版 | 部署配置（密码替换为 `***`）| YAML |
| 运维手册 | 常用操作命令（重启/查日志/扩容）| Markdown |
| 版本清单 | 各组件版本号 + 镜像摘要 | Markdown |
