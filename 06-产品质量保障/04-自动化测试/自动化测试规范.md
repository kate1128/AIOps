# 自动化测试规范

> 自动化测试是质量保障的"守门员"——在 CI 流水线中自动运行，快速反馈代码变更的影响。

---

## 一、自动化测试层级

| 层级 | 框架推荐 | 运行阶段 | 运行频率 |
|------|----------|----------|----------|
| **单元测试** | Pytest / JUnit / Jest | CI 提交阶段 | 每次提交 |
| **集成测试** | Pytest + Testcontainer / Spring Boot Test | CI 集成阶段 | 每次 MR |
| **API 测试** | Postman / Newman / REST Assured | CI 部署后 | 每次部署 |
| **E2E 测试** | Playwright / Cypress / Selenium | 预发部署后 | 每次发布 |
| **UI 快照测试** | Storybook / Chromatic | CI 提交阶段 | 每次提交 |

---

## 二、测试代码规范

### 命名规范

```
test_{模块}_{场景}_{期望结果}

# 示例
test_user_login_with_valid_credentials_should_succeed
test_user_login_with_expired_password_should_fail
```

### 断言规范

- 每个测试用例只测试一个行为
- 使用明确的断言消息
- 避免多重断言

### 数据管理

- 测试数据使用 Fixture 或 Factory 生成
- 不依赖共享的测试数据
- 测试完成后清理数据

## 三、CI 集成要求

| 阶段 | 测试类型 | 阻断条件 |
|------|----------|----------|
| Commit | 单元测试 | 失败阻断 |
| MR | 单元 + 集成测试 | 失败阻断 |
| 部署 dev | API 测试 | 失败阻断 |
| 部署 pre | E2E 测试 | 告警不阻断 |
| 部署 prod | 监控断言 | 告警触发回滚 |

### CI 测试超时设置

| 测试类型 | 超时时间 |
|----------|----------|
| 单元测试 | 5 分钟 |
| 集成测试 | 15 分钟 |
| API 测试 | 10 分钟 |
| E2E 测试 | 30 分钟 |

---

## 四、质量门禁

| 门禁 | 阈值 | 处理 |
|------|------|------|
| 单元测试通过率 | 100% | 不达标阻断 MR |
| 单元测试覆盖率（核心模块）| > 80% | 不达标告警 |
| 单元测试覆盖率（整体）| > 60% | 不达标告警 |
| E2E 测试通过率 | > 95% | 不达标告警 |

---

## 五、Flaky Test 处理

| 场景 | 处理方式 |
|------|----------|
| 偶尔失败的测试 | 标记为 flaky，不阻断 CI |
| 连续 3 次失败 | 禁用该测试，创建 Bug 单跟踪 |
| 根因定位 | 修复后移除 flaky 标记 |
| Flaky 率超 5% | 全量测试 Review，排查系统性问题 |
