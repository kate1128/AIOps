# Semantic Release

> 完全自动化的语义化发布工具，从 Commit 分析 → 版本号 → CHANGELOG → 发布全流程自动化。

---

## 适用场景

- Node.js / JavaScript 技术栈
- 追求"Commit 即发布"的极致自动化
- GitHub Actions 生态

---

## 优劣势

| 优势 | 劣势 |
|------|------|
| 全流程自动化 | Node 生态，其他语言需插件 |
| 插件机制灵活 | 学习曲线较陡 |
| GitHub 生态深度集成 | GitLab 支持较弱 |
| 成熟的社区生态 | 高度自动化也意味着灵活性降低 |
