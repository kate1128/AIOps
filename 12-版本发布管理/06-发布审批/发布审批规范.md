# 发布审批规范

> 发布审批是生产安全的"门禁"——确保每次上线都经过必要的审核，同时保持审批效率。

---

## 一、审批矩阵

### 按发布类型和环境

| 发布类型 | dev | pre | prod（灰度） | prod（全量） |
|----------|-----|-----|-------------|-------------|
| **常规发布** | 自动 | 技术负责人 | - | 技术负责人 |
| **紧急发布（P0）** | 自动 | 值班人员 | - | 值班人员 |
| **紧急发布（P1）** | 自动 | 值班人员 | - | 值班人员 |
| **回滚发布** | 自动 | 值班人员 | - | 值班人员 |
| **配置变更** | 自动 | 技术负责人 | - | 技术负责人+审批人 |

### 按变更风险

| 风险等级 | 定义 | 审批要求 |
|----------|------|----------|
| **低风险** | 日志配置、监控阈值、UI 文案 | 无需审批 |
| **中风险** | 非核心服务更新、配置参数调整 | 技术负责人审批 |
| **高风险** | 核心服务更新、数据迁移、API 变更 | 技术负责人 + 产品负责人审批 |

---

## 二、审批流程

### 常规发布审批流

```
技术负责人 提交发布申请
    │
    ▼
审批人 收到审批通知
    │
    ├── ✅ 批准
    │     └→ CI 自动执行发布流程
    │
    └── ❌ 拒绝
          └→ 填写拒绝原因，退回修改
```

### 审批内容

审批人需确认以下事项：
- [ ] 发布内容是否经过测试验证
- [ ] 安全扫描是否通过
- [ ] 回滚预案是否就绪
- [ ] 变更日志是否完整
- [ ] 是否按规范进行了灰度验证

---

## 三、审批超时机制

| 审批类型 | 超时时间 | 超时处理 |
|----------|----------|----------|
| 常规发布 | 4 小时 | 自动提醒审批人 |
| 紧急发布 | 30 分钟 | 自动升级到上一级审批人 |
| 回滚发布 | 15 分钟 | 默认批准 |

---

## 四、审批记录

每次审批自动记录以下信息用于审计：

| 字段 | 说明 |
|------|------|
| 发布时间 | 精确到秒 |
| 发布版本 | 具体版本号 |
| 发布人 | 提交发布的人 |
| 审批人 | 审批通过的人 |
| 变更摘要 | 本次发布的核心变更内容 |
| 审批结果 | 批准 / 拒绝 |
| 拒绝原因 | 拒绝时的原因说明 |

---

## 五、禁止规则

| ❌ 禁止 | 原因 |
|---------|------|
| 跳过审批直接发布 | 绕过质量门禁 |
| 审批人自己和发布人同一人 | 失去审批制衡作用 |
| 使用审批人账号代批 | 审计无法追踪 |
| 高频紧急发布绕过审批 | 紧急发布应报备，事后复盘 |
