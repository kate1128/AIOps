# 变更日志规范

> 变更日志（CHANGELOG）是版本的"说明书"——让用户和团队快速了解每个版本改了什么。

---

## 一、Commit 类型约定（Conventional Commits）

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Commit 类型

| 类型 | 说明 | CHANGELOG 分组 |
|------|------|---------------|
| `feat` | 新功能 | 🚀 新功能 |
| `fix` | Bug 修复 | 🐛 Bug 修复 |
| `docs` | 文档变更 | 📝 文档变更 |
| `style` | 格式调整（不影响语义）| 不进入 CHANGELOG |
| `refactor` | 重构 | 🔧 重构 |
| `perf` | 性能优化 | ⚡ 性能优化 |
| `test` | 测试相关 | 不进入 CHANGELOG |
| `chore` | 构建/CI/工具变更 | 不进入 CHANGELOG |
| `break` | 不兼容变更 | 💥 不兼容变更 |

### 示例

```
feat(api): 添加用户批量导入接口 PROJ-123
fix(web): 修复 Safari 浏览器下页面崩溃问题 PROJ-456
break(core): 重构认证模块，移除旧版 JWT 支持 PROJ-789
```

## 二、CHANGELOG 模板

```markdown
# Changelog

## [v1.1.0] - 2024-01-15

### 💥 不兼容变更
- 移除已废弃的 v1 API（改用 v2）

### 🚀 新功能
- 用户批量导入接口
- 仪表盘数据导出 CSV 功能

### 🐛 Bug 修复
- 修复 Safari 浏览器下页面崩溃问题
- 修复夜间模式下通知弹窗样式错乱

### ⚡ 性能优化
- 优化首页加载速度（首屏时间从 3s 降到 1.2s）

### 🔧 重构
- 重构用户认证模块，减少重复代码

### 📝 文档
- 补充 API 文档中批量导入接口的说明

---

## [v1.0.0] - 2024-01-01

### 🚀 新功能
- 首次正式发布
```

---

## 三、CHANGELOG 规范

| 规则 | 说明 |
|------|------|
| **每个版本一个条目** | 按时间倒序排列，最新在上 |
| **版本号 + 发布日期** | 格式 `[vX.Y.Z] - YYYY-MM-DD` |
| **按类型分组** | feat / fix / perf / break 等 |
| **每个条目关联 Issue** | 格式 `PROJ-123` 或 `#123` |
| **版本对比链接** | 在文件末尾提供版本 diff 链接 |

## 四、自动化生成

### ReleasePlease（推荐）

通过 Git 历史中的 Conventional Commits 自动生成 CHANGELOG：

```
# release-please-config.json
{
  "packages": {
    ".": {
      "release-type": "simple",
      "changelog-sections": [
        {"type": "feat", "section": "🚀 新功能"},
        {"type": "fix", "section": "🐛 Bug 修复"},
        {"type": "perf", "section": "⚡ 性能优化"},
        {"type": "refactor", "section": "🔧 重构"},
        {"type": "break", "section": "💥 不兼容变更"},
        {"type": "docs", "section": "📝 文档变更"}
      ]
    }
  }
}
```

### 手动补充

自动生成的 CHANGELOG 可能缺少上下文，建议发布前人工审核补充：
- 补充影响范围说明
- 补充升级注意事项
- 补充已知问题
