# 环境同步规范

> 多环境（dev / pre / prod）的配置、版本、数据同步策略。核心理念：同一份制品+差异化配置。

---

## 一、同步原则

| 原则 | 说明 |
|------|------|
| **同一份制品** | dev / pre / prod 使用同一镜像 Tag，不做环境差异化构建 |
| **配置分离** | 配置和环境绑定，不硬编码在代码中 |
| **数据隔离** | pre 不得访问 prod 数据库 |
| **密钥独立** | 每个环境使用独立密钥和证书 |

---

## 二、同步维度

| 维度 | 同步策略 | 工具/机制 |
|------|----------|----------|
| **代码版本** | dev = 最新开发分支, pre/prod = Release Tag | Git branching |
| **容器镜像** | 同一 Tag 在各环境复用 | Harbor |
| **配置文件** | 环境变量 / ConfigMap / 配置中心 | Helm Values / K8s ConfigMap |
| **密钥证书** | 各环境独立，从 Vault 分发 | Vault / Sealed Secrets |
| **数据库 Schema** | 自动迁移脚本，版本控制 | Flyway / Liquibase / Alembic |
| **静态资源** | 同一 CDN 链接，缓存策略不同 | CDN / 对象存储 |

---

## 三、配置管理

### 推荐方案：Helm Values 分层

```
charts/
├── values.yaml            # 公共默认值
├── values-dev.yaml        # dev 环境覆盖
├── values-pre.yaml        # pre 环境覆盖
└── values-prod.yaml       # prod 环境覆盖
```

### 禁止做法

| ❌ 禁止 | 原因 |
|---------|------|
| 代码中硬编码环境地址 | 环境切换需重新构建 |
| pre 环境使用 prod 数据库 | 数据污染风险 |
| 手动修改生产配置 | 无审计、易出错 |
| 配置文件和镜像绑定 | 配置变更需重新构建 |

---

## 四、环境差异矩阵

| 项目 | dev | pre | prod |
|------|-----|-----|------|
| 代码版本 | 最新分支 | Release Tag | Release Tag |
| 镜像 | 开发构建 | 正式构建 | 正式构建（相同 Tag）|
| GPU 资源 | 共享 | 共享 | 独占 |
| 数据存储 | 测试数据 | 脱敏生产数据 | 生产数据 |
| 日志级别 | DEBUG | INFO | INFO |
| 监控告警 | 基础 | 完整 | 完整 + 值班 |
| 外部依赖 | Mock / 沙箱 | 沙箱 / 测试 | 生产 |
