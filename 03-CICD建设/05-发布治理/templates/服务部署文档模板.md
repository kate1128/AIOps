## 4. 部署步骤

### 4.1 K8s / ArgoCD

1. 更新镜像 Tag 或配置仓库版本
2. 执行 ArgoCD 同步或等待自动同步
3. 检查 Pod 滚动升级和 readiness 状态
4. 记录实际发布时间和执行人

### 4.2 Docker Compose

1. 登录目标服务器
2. 更新 `IMAGE_TAG` 或部署包版本
3. 执行 `docker compose pull` 和 `docker compose up -d`
4. 检查容器状态和 healthcheck

### 4.3 二进制 / SSH

1. 下载新版本 JAR 或二进制包
2. 停止旧进程并备份当前版本
3. 启动新版本并检查健康状态
4. 记录执行命令和机器范围
