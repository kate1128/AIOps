# 安全扫描

安全扫描是 CI/CD 流水线中的**质量门禁**，在制品进入部署阶段前拦截已知漏洞和不合规镜像。覆盖三个层面：依赖漏洞（CVE）、镜像来源可信（签名）、物料清单可追溯（SBOM）。

---

| 项目 | 状态 | 说明 |
|---|---|---|

| 镜像签名（Cosign）| ❌ 待接入 | 无签名机制，无法验证镜像来源 |

> **当前风险**：任何人推送到 Harbor 的镜像都可以被部署，没有来源验证。

## 扫描流程

```
CI build 产出镜像
        │
        ▼
┌──────────────────────────────────────────┐
│  Step 1: Trivy 漏洞扫描                   │
│  trivy image harbor.internal/xxx:tag     │
│                                          │
│  ├── CRITICAL 漏洞 → 阻断流水线           │
│  ├── HIGH 漏洞 → 告警 + 人工确认          │
│  └── MEDIUM/LOW → 记录，不阻断            │
└──────────────────────────────────────────┘
        │ ✅ 无 CRITICAL
        ▼
┌──────────────────────────────────────────┐
│  Step 2: Syft 生成 SBOM                  │
│  syft harbor.internal/xxx:tag            │
│  → 输出 sbom.json（软件物料清单）          │
│  → 上传到 Harbor / MinIO 归档             │
└──────────────────────────────────────────┘
        │ ✅
        ▼
┌──────────────────────────────────────────┐
│  Step 3: Cosign 镜像签名                  │
│  cosign sign harbor.internal/xxx:tag     │
│  → 签名附在 Harbor 镜像 manifest 上       │
└──────────────────────────────────────────┘
        │ ✅ 签名完成
        ▼
     进入 CD 部署阶段（04-持续部署/）
     ArgoCD 部署前验证签名（cosign verify）
```

## GitLab CI 集成配置

```yaml
# .gitlab-ci.yml
stages:
  - build
  - scan        # ← 安全扫描阶段，在 build 之后，deploy 之前
  - deploy

trivy-scan:
  stage: scan
  image:
    name: aquasec/trivy:latest
    entrypoint: [""]
  script:
    - trivy image
        --exit-code 1               # CRITICAL 时返回非 0，阻断流水线
        --severity CRITICAL,HIGH
        --ignore-unfixed            # 忽略暂无修复版本的漏洞
        --format table
        $IMAGE_NAME:$CI_COMMIT_SHORT_SHA
  allow_failure: false              # 必须通过，不允许跳过
  only:
    - main
    - tags

syft-sbom:
  stage: scan
  image: anchore/syft:latest
  script:
    - syft $IMAGE_NAME:$CI_COMMIT_SHORT_SHA -o json > sbom.json
  artifacts:
    paths:
      - sbom.json
    expire_in: 90 days

cosign-sign:
  stage: scan
  image: gcr.io/projectsigstore/cosign:latest
  script:
    - cosign sign --key $COSIGN_KEY $IMAGE_NAME:$CI_COMMIT_SHORT_SHA
  only:
    - main
    - tags
```

## 漏洞处置规范

| 严重程度 | 处置方式 | 时限 |
|---|---|---|
| **CRITICAL** | 必须修复，阻断部署 | 24 小时内 |
| **HIGH** | 需修复或添加白名单说明 | 1 周内 |
| **MEDIUM** | 记录，纳入下次迭代 | 1 个月内 |
| **LOW / UNKNOWN** | 记录，低优先级处理 | 按需 |

### 白名单配置（暂时豁免）

```yaml
# .trivyignore（放在项目根目录）
# 格式：CVE编号 + 豁免原因和截止日期
CVE-2023-12345  # 暂无修复版本，已知风险，评估后豁免，截止 2026-08-01
```

## Harbor 自动扫描配置

Harbor 可在镜像 push 时自动触发 Trivy 扫描（无需 CI 配置）：

```
Harbor → 项目设置 → 安全
  ☑ 自动扫描（推送时触发）
  ☑ 阻止漏洞严重性 >= Critical 的镜像被拉取
```

> 建议 CI 中扫描 + Harbor 自动扫描双重保障。
