## 替代方案一览

> 现有工具的替代方案对比，放在各分类 `未采用/` 目录中。

| 当前工具 | 替代方案 | 核心差异 | 引入能获得什么 |
|---------|---------|---------|----------------|
| **GitLab CI** | [Jenkins](./01-CI引擎/未采用/Jenkins.md) | Groovy 流水线，1800+ 插件 | 极灵活流水线、丰富插件生态 |
| **GitLab CI** | [GitHub Actions](./01-CI引擎/未采用/GitHubActions.md) | GitHub 原生，社区 Action 市场 | 零运维 Runner、矩阵构建 |
| **GitLab CI** | [Tekton](./01-CI引擎/未采用/Tekton.md) | K8s 原生 CRD 流水线 | 完全 K8s 原生、DAG 编排 |
| **Harbor** | [Docker Registry](./02-制品仓库/未采用/DockerRegistry.md) | 极简镜像仓库 | 极轻量、零依赖 |
| **ArgoCD** | [Flux](./03-CD部署/未采用/Flux.md) | 纯 Operator GitOps | 更轻量、Kustomize 支持好 |
| **ArgoCD** | [Spinnaker](./03-CD部署/未采用/Spinnaker.md) | 多云发布编排 | 精细的金丝雀/灰度策略 |
| **Trivy** | [Snyk](./04-安全扫描/未采用/Snyk.md) | 全栈安全 SaaS 平台 | 自动修复 PR、策略引擎 |
| **Trivy** | [Clair](./04-安全扫描/未采用/Clair.md) | Red Hat 专业扫描引擎 | Harbor 原生集成 |
| **Ansible** | [Terraform](./05-部署编排/未采用/Terraform.md) | IaC，云资源编排 | 云资源声明式管理、计划预览 |
| **Ansible** | [SaltStack](./05-部署编排/未采用/SaltStack.md) | Master-Minion 架构 | 执行速度快、万台级管理 |
