# 流量可观测

> 谁在调哪个 API、延迟多少、错误率如何——了解系统真实的使用模式。

---

## 当前能力

| 能力 | 状态 | 说明 |
|------|------|------|
| API 流量分析 | 🟡 基础 | Nginx access log 有，但未接入 Loki 做结构化分析 |
| 热点接口统计 | 🔴 未覆盖 | 无 Top N URI 请求量/延迟排名 |
| 状态码分布 | 🟡 基础 | Nginx 日志中有，但未可视化 |
| 客户端/地域分布 | 🔴 未覆盖 | 未做 IP 地理分析 |
| 流量录制与回放 | 🔴 未覆盖 | 无生产流量录制用于测试 |
| 网关指标 | 🟡 基础 | Nginx stub_status 可看基础连接数 |
| 慢请求追踪 | 🔴 未覆盖 | 无法定位慢请求的完整链路 |

## 推荐引入

1. **Nginx access log → Loki**（立即可做）
   - Promtail 采集 Nginx/Ingress access log
   - LogQL 分析：热点接口、错误率、请求时段分布
   - Grafana 建 "API 流量" Dashboard

2. **Ingress Controller 指标**（K8s 立即可做）
   - nginx-ingress-controller 默认暴露 10254/metrics
   - 关键指标：`nginx_ingress_controller_requests`、`ingress_upstream_latency_seconds`

3. **GoReplay 录制回放**（中期）
   - 录制生产请求到文件
   - 回放到预发环境做压力测试/回归验证
