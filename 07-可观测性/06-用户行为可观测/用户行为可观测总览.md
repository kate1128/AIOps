# 用户行为可观测

> 了解用户在产品中的真实行为路径——功能使用率、留存、转化、用户体验。

---

## 当前能力

| 能力 | 状态 | 说明 |
|------|------|------|
| 前端用户行为追踪 | 🔴 未覆盖 | 无页面浏览、点击流、会话录制的采集 |
| 产品数据分析 | 🔴 未覆盖 | 无法获取 DAU/MAU、功能使用率、留存率 |
| 用户满意度采集 | 🔴 未覆盖 | 无点赞/踩、评分反馈机制 |
| 会话回放 | 🔴 未覆盖 | 无法回放用户操作排查体验问题 |
| A/B 测试 | 🔴 未覆盖 | 无功能灰度发布和效果对比 |
| 转化漏斗 | 🔴 未覆盖 | 无用户从注册到核心行为的转化分析 |

## 推荐引入

1. **PostHog 自托管部署**（中期，业务需要时启动）
   - 自部署（4 容器左右），数据不出网
   - 覆盖：网页浏览、事件追踪、会话回放、A/B 测试、功能开关
2. **AI 服务反馈采集**（近期可做）
   - 在 vLLM API 加反馈端点（点赞/踩）
   - 数据存入 PostgreSQL，Grafana 展示
