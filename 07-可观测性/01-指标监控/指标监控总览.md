# 指标监控

> 系统的数字脉搏——按监测对象分四类：基础设施、中间件、应用服务、AI 推理。

---

## 分类总览

| 分类 | 监测对象 | 核心工具 | 文档 |
|------|---------|---------|------|
| **基础指标** | 主机 CPU/内存/磁盘/网络、容器、硬件、外部拨测 | node-exporter, Zabbix, Gatus | [基础指标/](./基础指标/) |
| **中间件指标** | Redis/PostgreSQL/Kafka/RabbitMQ/Nginx 等 16 种 | 各 Exporter | [中间件指标/](./中间件指标/) |
| **服务指标** | 应用层 QPS、错误率、P99 延迟、JVM | Micrometer / JMX Exporter | [服务指标/](./服务指标/) |
| **AI 服务指标** | GPU 显存/利用率、推理延迟、Token 用量 | DCGM Exporter, vLLM /metrics | [AI服务指标/](./AI服务指标/) |

---

## 推荐引入

1. **补齐 Exporter 部署**：非 K8s 节点部署 node-exporter，Docker 宿主机部署 cAdvisor
2. **各中间件 Exporter**：参见[中间件指标/](./中间件指标/)下各文档
3. **Java 服务 Micrometer**：二进制 Java 服务引入 Micrometer 暴露 /actuator/prometheus
4. **DCGM Exporter**：所有 GPU 节点部署，采集温度/显存/ECC 错误
