# 可观测工具总览

> 可观测性（Observability）由三大支柱组成：**指标（Metrics）**、**日志（Logs）**、**链路追踪（Traces）**。本文梳理 SmartVision 项目评估过的全部工具，按采集 → 存储 → 可视化 → 告警 → 分析 的完整链路分类，给出选型结论。

---

## 六、方案对比：三大主流可观测栈

### 6.1 Grafana 全栈

```
采集：Alloy / OTel SDK
指标：Prometheus → Grafana
日志：Loki → Grafana
链路：Tempo → Grafana
告警：Alertmanager → PrometheusAlert → 飞书
```

| 优势 | 劣势 |
|------|------|
| 三维联动（指标→日志→链路跳转） | Loki 全文检索弱于 ES |
| 统一可视化（Grafana） | Tempo 生产案例少于 Jaeger |
| CNCF 生态原生 | 配置复杂度较高 |
| 存储成本低（对象存储） | — |

### 6.2 ELK 全栈

```
采集：Filebeat / Fluentd
存储：Elasticsearch
可视化：Kibana
告警：ElastAlert / Kibana Alerting
```

| 优势 | 劣势 |
|------|------|
| 全文检索能力极强 | 存储成本高（5-10x Loki） |
| 生产案例丰富 | JVM 调优运维负担重 |
| Kibana 功能成熟 | 无原生链路追踪 |

### 6.3 Netdata 独立方案

```
采集+存储+可视化+告警：Netdata（单 binary）
云端聚合：Netdata Cloud
```

| 优势 | 劣势 |
|------|------|
| 零配置、秒级精度 | 缺乏多节点统一视图 |
| 800+ 采集器内置 | 不支持飞书告警 |
| 内置 ML 异常检测 | 与 Prometheus 集成有限 |

## 七、完整告警链路

```
                Prometheus
                    │
                    │ 触发告警规则
                    ▼
              Alertmanager
              ┌─────┴─────┐
              │ 路由/分组   │
              │ 抑制/静默   │
              └─────┬─────┘
                    │ Webhook
                    ▼
          PrometheusAlertCenter
          ┌─────────┴─────────┐
          │ Go Template 渲染   │
          │ 飞书富文本卡片      │
          └─────────┬─────────┘
                    │
                    ▼
                 飞书群
              (OnCall 通知)
```

### 告警规则分类

| 类别 | 来源 | 示例 |
|------|------|------|
| 基础设施 | node-exporter / cAdvisor | CPU/内存/磁盘/网络 |
| K8s 对象 | kube-state-metrics | Pod Pending/Failed、Deployment 副本 |
| 中间件 | 各 Exporter | Redis 内存、PG 连接数、Kafka Lag |
| 应用层 | OTel SDK / 自定义 | HTTP 延迟、错误率、QPS |
| GPU | DCGM Exporter | 温度、显存、利用率、ECC 错误 |
| AI 推理 | vLLM 内置 | TTFT、KV Cache、请求队列 |

## 八、技术选型决策树

```
需要可观测性？
├── 指标监控
│   ├── K8s 环境 → kube-prometheus-stack（一键部署）
│   ├── 小团队/从零 → Netdata（零配置）
│   └── 长期存储 → VictoriaMetrics（比 Thanos 简单）
├── 日志聚合
│   ├── Grafana 栈 → Loki（成本最低）
│   ├── 需全文检索 → Elasticsearch（功能最强）
│   └── 资源受限 → Fluent Bit（5MB 内存）
├── 链路追踪
│   ├── Grafana 栈 → Tempo（三维联动）
│   ├── Java 微服务 → SkyWalking（All-in-One）
│   └── 通用场景 → Jaeger（社区最广）
├── 错误追踪
│   ├── 全功能 → Sentry（最强但运维重）
│   └── 轻量级 → PostHog（已覆盖）
├── 用户行为
│   ├── 全功能 → PostHog（首选）
│   ├── 极简隐私 → Plausible
│   └── 传统 Web → Matomo
└── 告警通知
    ├── Prometheus 栈 → Alertmanager + PrometheusAlert
    └── Grafana 栈 → Grafana Alerting
```
